### Commit f955a0e — feat(rollback): resolve previous image tag from gitops history
- **Date:** 2026-05-02

  `docs/guides/rollbacks.md` (add a section on agent-triggered rollback).

### Commit 6ce1058 — Wire diagnostics through the core chat command (#72936)
- **Date:** 2026-04-28
- **Type:** feat (three bundled user-visible capabilities)
- **User-visible effect:** Three related capabilities were shipped in one
  PR that was missed by the 2026-04-29 scan (GITHUB_TOKEN was absent that
  day):

  1. **`/diagnostics` slash command**: owners can now type `/diagnostics`
     in any channel chat to trigger a local Gateway diagnostics export
     without leaving the conversation. OpenClaw requests explicit exec
     approval before running `openclaw gateway diagnostics export --json`,
     then replies with the bundle path, manifest summary, and privacy
     notes. In group chats the sensitive output is routed to the owner
     privately.

  2. **`sessions export-trajectory` subcommand**: a new CLI subcommand
     `openclaw sessions export-trajectory --session-key <key>` exports a
     redacted trajectory bundle for a stored session. This is the command
     path triggered by the `/export-trajectory` slash command after owner
     approval.

  3. **Pairing owner bootstrap**: approving a DM pairing code now also
     sets `commands.ownerAllowFrom` to the approved sender if no command
     owner is configured yet, giving first-time setups an automatic owner
     for privileged commands without a separate config step.

- **Docs:** gap — `docs/platform/openclaw.md` covers OpenClaw at the
  integration level but has no section on owner-privileged slash commands
  (`/diagnostics`, `/export-trajectory`) or the pairing owner-bootstrap
  behaviour. `docs/reference/troubleshooting.md` could also reference the
  diagnostics command as a first-step debugging action.
- **Suggested doc location:** `docs/platform/openclaw.md` — add a
  "Privileged commands" or "Diagnostics and session exports" section
  covering `/diagnostics`, `/export-trajectory`, and first-owner bootstrap
  via pairing approval. Optionally cross-reference from
  `docs/reference/troubleshooting.md`.
