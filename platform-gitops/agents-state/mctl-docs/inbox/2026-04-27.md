## External: CVE / Security Advisories

### CVE-2026-39363 — Vite arbitrary file read via dev server WebSocket
- **Affects:** Vite (dep of VitePress) — dev server mode (`vitepress dev`)
- **Description:** missing access control in the Vite dev server WebSocket allows reading arbitrary files when the dev server is explicitly exposed on the network and `server.ws` is disabled. Patched in Vite v6.4.2, v7.3.2, v8.0.5.
- **User-visible effect:** the vulnerability concerns only dev mode (contributors running `vitepress dev` with `--host`). Production builds (static HTML) are not affected. docs.mctl.ai users are safe; contributors are at risk if they run dev with network exposure in CI/CD or on public servers.
- **Docs:** gap — there is no contributing guide and no security note about safely running the dev environment.
- **Suggested doc location:** new page `docs/reference/contributing.md` with a section "Running the dev server safely".

### CVE-2026-4800 & CVE-2026-2950 — lodash-es (mermaid dependency)
- **Affects:** lodash-es 4.x (dependency of mermaid 11.x); patched in lodash-es 4.18.1, but mermaid has not yet updated the dep as of the run date.
- **Description:** vulnerabilities in lodash-es, a transitive dependency of mermaid. Specific attack vectors for CVE-2026-4800/2026-2950 are not disclosed in public advisories as of 2026-04-27.
- **User-visible effect:** the mermaid bundle in docs.mctl.ai includes vulnerable lodash-es. If the attack vector is XSS via the mermaid renderer (since all content is static and there is no user input, real risk is low), the site nevertheless technically contains vulnerable code.
- **Docs:** gap — docs.mctl.ai has no security.txt or dependency changelog. The infra decision (bump mermaid or pin lodash-es) needs to be documented in an ADR.
- **Suggested doc location:** `context/decisions/0003-mermaid-dep-security.md` (internal ADR) + optional security.txt in `docs/public/`.

### CVE-2026-23733 — mermaid XSS in LobeChat (RCE in Electron)
- **Affects:** LobeChat (uses mermaid), not mermaid core directly.
- **User-visible effect for mctl-docs:** minimal — docs.mctl.ai is not an Electron application and does not render user-supplied mermaid. Diagram content is controlled by the authors of doc PRs.
- **Docs:** documented (risk not applicable).
