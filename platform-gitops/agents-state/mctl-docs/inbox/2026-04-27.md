## External: CVE / Security Advisories

### CVE-2026-39363 — Vite arbitrary file read via dev server WebSocket
- **Затрагивает:** Vite (dep VitePress) — dev server режим (`vitepress dev`)
- **Описание:** отсутствие access control в Vite dev server WebSocket позволяет читать произвольные файлы при явном сетевом экспозировании dev-сервера и отключённом `server.ws`. Patched в Vite v6.4.2, v7.3.2, v8.0.5.
- **User-visible effect:** уязвимость касается только dev mode (contributors running `vitepress dev` с `--host`). Продакшн-сборка (static HTML) не затронута. Пользователи docs.mctl.ai в безопасности; под угрозой contributors если запускают dev с network exposure в CI/CD или на публичных серверах.
- **Доки:** gap — нет contributing guide и нет security-заметки о безопасности запуска dev-среды.
- **Suggested doc location:** новая страница `docs/reference/contributing.md` с секцией "Running the dev server safely".

### CVE-2026-4800 & CVE-2026-2950 — lodash-es (mermaid dependency)
- **Затрагивает:** lodash-es 4.x (dependency mermaid 11.x); patched в lodash-es 4.18.1, но mermaid ещё не обновил dep на дату прогона.
- **Описание:** уязвимости в lodash-es, транзитивной зависимости mermaid. Конкретные векторы CVE-2026-4800/2026-2950 не раскрыты в public advisories на 2026-04-27.
- **User-visible effect:** mermaid bundle в docs.mctl.ai включает уязвимый lodash-es. Если вектор атаки — XSS через mermaid renderer (т.к. весь контент static и нет user input, реальный риск низкий), но технически сайт содержит уязвимый код.
- **Доки:** gap — docs.mctl.ai не имеет security.txt или changelog о зависимостях. Инфра-решение (bump mermaid или pin lodash-es) нужно задокументировать в ADR.
- **Suggested doc location:** `context/decisions/0003-mermaid-dep-security.md` (внутренний ADR) + при необходимости security.txt в `docs/public/`.

### CVE-2026-23733 — mermaid XSS in LobeChat (RCE в Electron)
- **Затрагивает:** LobeChat (использует mermaid), не mermaid core напрямую.
- **User-visible effect для mctl-docs:** минимальный — docs.mctl.ai не является Electron-приложением и не рендерит user-supplied mermaid. Контент диаграмм контролируется авторами doc PR.
- **Доки:** documented (риск неприменим).
