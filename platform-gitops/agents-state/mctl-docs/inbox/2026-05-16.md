### Commit `5b166aa` — fix(auth): add retry grace window and client-scoped revocation to refresh store
- **Date:** 2026-05-13
- **Type:** fix
- **User-visible effect:** Token revocation via the OAuth `/revoke` endpoint
  now accepts a `client_id` parameter (per RFC 7009 §2.1), scoping revocation
  to tokens issued for that specific client. Additionally, a retry grace window
  prevents race conditions where a legitimate rotation attempt on an already-
  rotated token would be incorrectly rejected. Users of multi-client OAuth
  setups (e.g. separate MCP client + web client) will see more precise
  revocation behaviour without accidental cross-client token invalidation.
- **Docs:** gap — `docs/security/authentication.md` does not document the
