### Commit `41b6d4d` — feat(agent-runs): include cron-driven workflows in mctl_list_recent_agent_runs
- **Date:** 2026-05-07
- **Type:** feat
- **User-visible effect:** The MCP tool `mctl_list_recent_agent_runs` now
  merges cron-driven workflow runs (mctl-agents-daily, mctl-agents-shepherd,
  mctl-agents-implement, etc.) with operator-initiated runs into a single
  unified list sorted by timestamp descending, capped at 10. Each item now
  carries a new `source` field (`"operator"` | `"cron"`) so callers can

  fired every morning.
- **Docs:** gap — `docs/mcp/tools-reference.md` documents
  `mctl_list_recent_agent_runs` but does not reflect the merged cron+operator
  view or the new `source` field in each item.
- **Suggested doc location:** `docs/mcp/tools-reference.md` — update the
  `mctl_list_recent_agent_runs` entry to document: (1) merged cron+operator
  list, (2) `source` field semantics, (3) example response showing both
  source types.
- **Files touched:** `internal/api/handlers_read.go` (+37/-6),
  `internal/api/interfaces.go` (+7/-0),
  `internal/operations/executor.go` (+89/-0),
  `internal/mcp/server.go` (+1/-1),
  `internal/api/workflow_read_test.go` (+72/-0).
- **Diff highlight:** New `Executor.ListCronAgentRuns(ns, prefix, since)`

### Commit `b73a825` — fix(agent-runs): normalize operator timestamp to RFC3339 string
- **Date:** 2026-05-07
- **Type:** fix

  coherent story.

### Commit `74ee766` — feat(skill): argocd_sync_failed — diagnose stuck-degraded ArgoCD apps
- **Date:** 2026-05-06
- **Type:** feat
- **User-visible effect:** A new built-in skill (`argocd_sync_failed`) is now
  active in the self-healing agent. When an ArgoCD application enters an
  `OutOfSync+Degraded` or `Degraded` state (including after an alert from
  AlertManager), the agent automatically diagnoses the failure and posts
  recovery commands to Telegram for human approval. It recognises two known
  recovery patterns: (1) CRD storedVersion conflicts ("must remain in
  spec.versions") and (2) managedFields poisoning from a CRD recovery window.
  The skill diagnoses only — `Fix()` is nil — so no unsupervised cluster
  changes occur. Platform admins who relied on mctl-agent to detect ArgoCD
  issues will now see actionable remediation advice in Telegram rather than
  silent ticket accumulation.
- **Docs:** gap — `docs/platform/components.md` covers mctl-agent and its
  existing skills but the new `argocd_sync_failed` skill is undocumented. The
  component page likely lists only the rollback skill as an example.
- **Suggested doc location:** `docs/platform/components.md` — add a
  subsection or table entry for the `argocd_sync_failed` diagnostic skill,
  describing the trigger conditions, what gets posted to Telegram, and the
  manual-approval model.
- **Files touched:** `internal/skill/builtin/argocd_sync_failed.go` (+151/-0),
  `internal/monitor/alerthandler.go` (+2/-0),
