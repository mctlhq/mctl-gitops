## Repo: mctl-openclaw  (prod-version: 2026.5.2-mctl.1 — confirmed via mctl-gitops `532bd16` on 2026-05-04)

### Commit 0fad53a — feat(whatsapp): support newsletter targets in message tool
- **Date:** 2026-05-02
- **Type:** feat
- **User-visible effect:** The WhatsApp channel's `message send` tool now accepts
  WhatsApp Channel/Newsletter JIDs (`...@newsletter`) as explicit outbound targets.
  Previously only DM (E.164) and group (`@g.us`) targets were supported; newsletter
  sends would either silently fail or fall through the wrong routing path. The PR
  maps newsletter JIDs to a dedicated session key
  (`agent:<agentId>:whatsapp:channel:<jid>`) instead of treating them as DMs, and
  adds a clarification that `allowFrom` is a DM-only access-control list (it does
  not gate outbound sends to `@newsletter` or `@g.us` JIDs).
- **Docs:** gap — `docs/platform/openclaw.md` describes the OpenClaw integration but
  has no section on outbound target formats for specific channels. The ID format
  `...@newsletter` is now valid but is not mentioned anywhere in `docs.mctl.ai`.
  The `mctl-openclaw` repo itself updated `docs/channels/whatsapp.md`,
  `docs/cli/directory.md`, and `docs/cli/message.md` with these details, giving the
  spec-writer authoritative source material.
- **Suggested doc location:** `docs/platform/openclaw.md` — add a "WhatsApp outbound
  targets" paragraph or table listing DM (`+15551234567`), group (`@g.us`), and
  Channel/Newsletter (`@newsletter`) formats, noting the `allowFrom` scope restriction.
- **Diff highlight (mctl-openclaw docs/channels/whatsapp.md):**
  ```diff
  +- WhatsApp Channels/Newsletters can be explicit outbound targets with their native
  +  `@newsletter` JID. Outbound newsletter sends use channel session metadata
  +  (`agent:<agentId>:whatsapp:channel:<jid>`) rather than DM session semantics.
  +
  +`allowFrom` is a DM sender access-control list. It does not gate explicit outbound
  +sends to WhatsApp group JIDs or `@newsletter` channel JIDs.
  ```
  And in `docs/cli/directory.md`:
  ```diff
  -- WhatsApp: `+15551234567` (DM), `1234567890-1234567890@g.us` (group)
  ++ WhatsApp: `+15551234567` (DM), `1234567890-1234567890@g.us` (group),
  +  `120363123456789@newsletter` (Channel/Newsletter outbound target)
  ```
- **Version status:** confirmed in production (mctl-openclaw 2026.5.2-mctl.1 deployed
  via mctl-gitops `532bd16` on 2026-05-04).
