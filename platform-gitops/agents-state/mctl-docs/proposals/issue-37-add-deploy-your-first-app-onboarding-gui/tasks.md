The change consists of one new file (`docs/guides/deploy-first-app.md`) and one additive line in `docs/.vitepress/config.ts`. Both can be reverted with a single `git revert <merge-commit>` on the default branch, which removes the new file and restores the sidebar array to its previous state. No data migration, no environment variable change, and no external service configuration is involved. After the revert, the next Docker build and nginx deployment will serve the previous static output without the new page; no cache invalidation is required beyond the standard CDN TTL.
