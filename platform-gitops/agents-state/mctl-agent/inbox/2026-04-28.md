## Top-3 (for spec-writer)

### 1. go-runtime-upgrade: Upgrade Go toolchain from 1.24 to 1.26.2
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Go 1.24 is behind both maintained release branches and carries 6 confirmed CVEs: two crypto/x509 DoS vectors (CVE-2026-32280, CVE-2026-32281), html/template XSS, archive/tar unbounded memory allocation, os/syscall symlink escape (CVE-2026-32282), and a compiler bounds bypass (CVE-2026-27144). Upgrading to Go 1.26.2 also subsumes CVE-2025-22871 (net/http HTTP request smuggling), eliminating the entire batch in a single toolchain bump with minimal code changes expected.
**Source:** https://groups.google.com/g/golang-announce/c/0uYbvbPZRWU; https://access.redhat.com/errata/RHSA-2026:10217

### 2. sqlite-cve-patch: Upgrade modernc.org/sqlite to resolve CVE-2025-6965
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2025-6965 is a CVSS 9.8 Critical memory corruption bug triggered when SQLite aggregate terms exceed available columns; the fix landed in SQLite 3.50.2, and modernc.org/sqlite 1.34 bundles an older SQLite version. The service uses SQLite for both the tickets database and skill metrics, meaning any path that runs aggregate queries against attacker-influenced data is a potential exploit surface. Bumping to modernc.org/sqlite 1.37.x (which ships SQLite 3.53.0) closes the vulnerability with no schema or API changes required.
**Source:** https://nvd.nist.gov/vuln/detail/CVE-2025-6965; https://pkg.go.dev/modernc.org/sqlite

### 3. alertmanager-webhook-audit: Audit and harden POST /api/v1/alerts against alertmanager v0.32.0 payload changes
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** alertmanager v0.32.0 (released 2026-04-08) introduced webhook payload templating, multiple matcher-set silences, and new template functions, any of which may have altered the JSON schema delivered to POST /api/v1/alerts. If the service silently misparses the new payload format, the entire self-healing pipeline stalls without an obvious error — routed alerts like PodCrashLooping and KubePodNotReady would stop triggering fixes. The audit should validate field mapping against the v0.32.0 webhook spec and add a schema-version guard or rejection log so regressions surface immediately.
**Source:** https://github.com/prometheus/alertmanager/releases/latest
