## Top-3 (for spec-writer)

### 1. go-toolchain-upgrade: Upgrade Go toolchain to v1.26.2
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** mctl-agent is built on Go 1.24 and is directly exposed to 7 CVEs fixed in Go 1.26.2, including CVE-2026-27140 (arbitrary code execution at build time via cmd/go) and CVE-2026-27143 (memory corruption in the compiler) — both critical severity. The remaining five CVEs cover TLS deadlock, memory exhaustion, x509 DoS (two variants), and a symlink race on Linux, all of which affect the runtime environment mctl-agent operates in. Upgrading the toolchain is a contained, low-effort change with no memory impact on the labs tenant.
**Source:** https://groups.google.com/g/golang-announce/c/0uYbvbPZRWU; CVEs GO-2026-4868, GO-2026-4869, GO-2026-4870, GO-2026-4871, GO-2026-4946, GO-2026-4947, CVE-2026-32282

### 2. go-github-authz-leak: Upgrade go-github to v85.0.0 (Authorization header leakage fix)
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** mctl-agent actively uses google/go-github (currently v68) to open fix PRs in mctlhq/mctl-gitops; v85.0.0 adds cross-host redirect rejection that prevents GitHub installation tokens from leaking via Authorization headers to attacker-controlled hosts. Token leakage would expose write access to the gitops repository, enabling unauthorized PR creation or branch modification. The upgrade involves breaking API changes (GetOrgRole, CreateCustomOrgRole, UpdateCustomOrgRole signatures and MarkThreadDone int64→string) that require a targeted migration pass.
**Source:** https://github.com/google/go-github/releases/latest (v85.0.0, 2026-04-20)

### 3. mcp-input-validation: Harden POST /mcp endpoint against command-injection
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** MCP command-injection advisories were publicly disclosed on 2026-04-15, and mctl-agent exposes a POST /mcp endpoint with 6 tools that interact with GitHub, AlertManager, and internal ticket state. Unsanitized tool arguments passed through the MCP JSON-RPC layer could enable prompt-injection or direct command-injection into downstream API calls. A structured input validation and allowlist schema for each of the 6 tools reduces the attack surface without architectural changes.
**Source:** MCP command-injection advisories disclosed 2026-04-15 (background context); mctl-agent architecture — POST /mcp with 6 tools
