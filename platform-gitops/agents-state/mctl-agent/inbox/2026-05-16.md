## Top-3 (for spec-writer)

### 1. go-toolchain-1263-patch: Upgrade Go toolchain to 1.26.3 for HTTP/2 and XSS CVE fixes
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Go 1.26.3 fixes five CVEs that are genuinely new and not covered by the existing `go-upgrade-1-26` proposal (which targets 1.26.2): CVE-2026-33814 (HTTP/2 infinite loop in net/http, directly exploitable against the chi-based webhook server), CVE-2026-39826 and CVE-2026-39823 (html/template XSS bypasses), and CVE-2026-42499/39820 (net/mail DoS). The service is still on Go 1.24, which has been out of active support since February 2026, making this upgrade both a security imperative and a compliance necessity. Toolchain upgrades carry low operational risk when no breaking Go-language changes are involved between patch versions.
**Source:** https://groups.google.com/g/golang-announce/c/qcCIEXso47M

### 2. go-github-v86-upgrade: Migrate google/go-github to v86.0.0 (context-via-Request API)
**Category:** refactor
**Impact:** 3
**Effort:** 4
**Rationale:** The service currently pins google/go-github at v68, which is 18 major versions behind v86.0.0; all existing proposals (`go-github-v85-upgrade`, `go-github-v85-migration`, etc.) stop at v85 and do not cover the v86 breaking API change where Context is passed via the Request rather than as a standalone parameter. Remaining on an ancient version of the GitHub client accumulates compounding migration debt and leaves OIDC auth improvements on the table. This work is isolated to the PR-creation path and does not affect the skill registry or SQLite schema, so risk is contained.
**Source:** https://github.com/google/go-github/releases/latest

### 3. anthropic-sdk-v143-upgrade: Upgrade anthropic-sdk-go to v1.43.0 for cache diagnostics and structured-output fix
**Category:** feature
**Impact:** 2
**Effort:** 1
**Rationale:** The existing `anthropic-sdk-v141-upgrade` proposal covers only v1.41.0; v1.43.0 adds cache diagnostics beta (useful for monitoring prompt-cache hit rates in the LLMDiagnosis skill) and fixes a structured-output schema transformation bug that could silently produce malformed tool-call payloads. The JSON encoder optimization is a free performance gain with no API surface changes. No memory footprint increase is introduced, so the `labs` tenant constraint is not affected.
**Source:** https://github.com/anthropics/anthropic-sdk-go/releases/latest
