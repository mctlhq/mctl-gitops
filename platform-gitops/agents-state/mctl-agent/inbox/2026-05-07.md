## Top-3 (for spec-writer)

### 1. argocd-secret-leak-cve: Patch platform Argo CD against CVE-2026-42880 (CVSS 9.6 plaintext secret leak)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Any authenticated read-only user on the platform can exfiltrate plaintext Kubernetes Secrets via the ServerSideDiff endpoint — a critical confidentiality breach that directly threatens mctl-agent's own Vault secrets and GitHub App credentials managed through Argo CD. The fix is a version bump to v3.3.9 or v3.2.11 with no API changes; effort is low relative to the blast radius. No existing proposal covers this CVE.
**Source:** https://cyberpress.org/critical-argo-cd-vulnerability/ — CVE-2026-42880, CVSS 9.6, inbox 2026-05-07.

### 2. argocd-cluster-label-drift: Update ArgoCDDrift skill for Argo CD v3.4.1 cluster version label format change
**Category:** refactor
**Impact:** 3
**Effort:** 2
**Rationale:** Argo CD v3.4.1 changed the ApplicationSet cluster generator version label format from "Major.Minor" to "vMajor.Minor.Patch"; the `ArgoCDDrift` builtin Go skill parses Argo CD Application and cluster resources, so any label-based version comparisons will silently mismatch after the platform upgrades Argo CD, producing false negatives on drift detection. Updating the skill's label parser and adding a table-driven test case closes the gap before the platform upgrade lands. No existing proposal covers this behavioral change.
**Source:** https://github.com/argoproj/argo-cd/releases/tag/v3.4.1 — Argo CD v3.4.1 release notes, inbox 2026-05-07.

### 3. anthropic-sdk-v141-upgrade: Upgrade anthropic-sdk-go to v1.41.0 for MCP tool helpers and webhook handling
**Category:** feature
**Impact:** 2
**Effort:** 2
**Rationale:** mctl-agent uses the Anthropic SDK for the diagnose phase and exposes its own MCP endpoint with 6 tools; v1.41.0 introduces first-class MCP tool helpers and webhook handling that can replace custom scaffolding in both areas, reducing maintenance surface. Managed-agents support also aligns with the architecture's remote-skill extensibility roadmap. The existing `anthropic-sdk-timeout-default` proposal addresses timeouts only and does not cover this SDK version upgrade. No memory increase is expected in `labs` tenant since this is a library swap, not additional in-process state.
**Source:** https://github.com/anthropics/anthropic-sdk-go/releases/tag/v1.41.0 — anthropic-sdk-go v1.41.0 release, inbox 2026-05-07.
