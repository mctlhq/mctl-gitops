## Top-3 (for spec-writer)

### 1. go-runtime-patch: Upgrade Go toolchain to 1.24.8 (security patch)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Ten CVEs were fixed in Go 1.24.8 / 1.25.2 (published 2026-04-07). The most critical are CVE-2026-27140 (arbitrary code execution via cmd/go SWIG at build time), CVE-2026-27143/27144 (compiler memory corruption), and CVE-2026-32283 (crypto/tls deadlock). The service is compiled with Go 1.24 and the fix is a toolchain-only bump — no API changes, minimal effort, maximum security gain. Direct impact on the mctl-agent binary.
**Source:** https://pkg.go.dev/vuln/list (go1.24.8 release, 2026-04-07)

### 2. go-github-upgrade: Upgrade google/go-github from v68 to v85
**Category:** security
**Impact:** 4
**Effort:** 4
**Rationale:** The service uses go-github to create fix PRs in mctl-gitops using a GitHub App installation token rotated every 30 minutes. Versions before v85 allow cross-host redirects that can leak the Authorization header, directly exposing the GitHub App credentials. The 17-major-version gap (v68→v85) also means accumulated bug fixes and API improvements are unavailable. Breaking changes are scoped to org-role and thread-ID APIs that mctl-agent likely does not use for PR creation, so migration risk is manageable.
**Source:** https://github.com/google/go-github/releases/tag/v85.0.0 (security fix for cross-host redirect auth token leakage)

### 3. alertmanager-webhook-v032: Validate and harden POST /api/v1/alerts handler for Alertmanager v0.32.x
**Category:** feature
**Impact:** 3
**Effort:** 2
**Rationale:** Alertmanager v0.32.0 (latest: v0.32.1) introduced webhook payload templating and multiple matcher set silences, potentially changing the structure of the JSON body posted to the service's `POST /api/v1/alerts` handler. If the platform upgrades Alertmanager and the handler silently misparses the new envelope, no alerts are processed and self-healing stops entirely. Low-effort defensive proposal: audit the parser against the v0.32.x schema, add a version-aware struct, and extend integration tests.
**Source:** https://github.com/prometheus/alertmanager/releases/tag/v0.32.1 (v0.32.0 changelog: webhook payload templating, multiple matcher set silences)
