## Top-3 (for spec-writer)

### 1. mcp-command-injection-hardening: Harden POST /mcp endpoint against MCP stdio command injection (CVE-2026-30623)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-30623 (published 2026-04-15) targets the MCP SDK stdio transport and mctl-agent is the only service in the platform that exposes a `POST /mcp` endpoint with 6 registered tools, making it a direct attack surface. Input validation, strict allowlisting of tool names, and request sanitization at the MCP handler layer can close the vector without a dependency swap. Addressing this proactively is warranted before the CVE is weaponised against the admins tenant.
**Source:** https://docs.litellm.ai/blog/mcp-stdio-command-injection-april-2026

### 2. go-github-v85-migration: Upgrade google/go-github from v68 to v85
**Category:** refactor
**Impact:** 3
**Effort:** 3
**Rationale:** mctl-agent depends on go-github for all GitHub PR creation flows (the core output of the fix pipeline); at v68 vs v85 the library is 17 major versions behind and the changelog documents breaking parameter-type changes that will cause silent misuse or compile errors if any intermediate upgrade is eventually forced. Migrating now while the PR-creation code is well-understood is cheaper than doing it under incident pressure. No memory impact on either tenant.
**Source:** https://github.com/google/go-github/releases (v85.0.0, released 2026-04-20)

### 3. chi-v5-security-upgrade: Upgrade go-chi/chi from v5.2.1 to v5.2.5 to adopt hardened RedirectSlashes middleware
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** Although mctl-agent on v5.2.1 is not currently vulnerable to CVE-2025-69725 (the open-redirect affects v5.2.2+), upgrading to v5.2.5 locks the service onto the hardened middleware implementation and eliminates the risk of accidentally sliding into the vulnerable range during any future patch. The upgrade is a drop-in patch with no API changes and negligible binary size delta — zero memory risk for either tenant.
**Source:** https://github.com/go-chi/chi/releases; https://www.sentinelone.com/vulnerability-database/cve-2025-69725/
