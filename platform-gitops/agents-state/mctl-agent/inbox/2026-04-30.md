## Top-3 (for spec-writer)

### 1. go-toolchain-upgrade: Upgrade Go toolchain from 1.24 to 1.26.2
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Six CVEs in the Go toolchain directly affect mctl-agent as compiled software: memory corruption in cmd/compile (CVE-2026-27143), x509 DoS (CVE-2026-32280), TLS 1.3 deadlock (CVE-2026-32283), XSS in html/template (CVE-2026-32289), SWIG RCE (CVE-2026-27140), and symlink race in os (CVE-2026-32282). mctl-agent is on Go 1.24 and receives no backport patches. Upgrading the toolchain closes all six vulnerabilities with minimal code changes — Go releases are backward-compatible within a major version, and this does not increase memory footprint for the labs tenant.
**Source:** https://groups.google.com/g/golang-announce/c/0uYbvbPZRWU; CVE-2026-27143, CVE-2026-32280, CVE-2026-32283, CVE-2026-32289, CVE-2026-27140, CVE-2026-32282

### 2. go-github-upgrade: Upgrade google/go-github from v68 to v85
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** go-github v85 adds cross-host redirect rejection that prevents Authorization header leakage — a direct threat given that mctl-agent uses this library to create fix PRs carrying a GitHub App installation token. At 17 major versions behind, there are API breaking changes to absorb (custom org role methods, ActivityService.MarkThreadDone signature), making this a medium-effort change, but the token-leakage risk on PR creation warrants priority. No memory impact on the labs tenant.
**Source:** https://github.com/google/go-github/releases/latest (v85.0.0, released 2025-04-20)

### 3. mcp-injection-audit: Audit POST /mcp endpoint for command injection (CVE-2026-22252, CVE-2026-30623)
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** A systemic command-injection advisory against Anthropic's MCP SDK STDIO interface was disclosed 2026-04-15 affecting Python, TypeScript, Java, and Rust variants; the Go SDK's exposure is unconfirmed but not ruled out. mctl-agent exposes POST /mcp with 6 tools, making it a viable target if the Go transport shares the same input-handling flaw. A focused audit of tool input sanitization and transport handling is low-effort relative to the potential RCE risk, and produces either a code fix or a documented safe-verdict — both valuable.
**Source:** https://docs.litellm.ai/blog/mcp-stdio-command-injection-april-2026 (CVE-2026-22252, CVE-2026-30623)
