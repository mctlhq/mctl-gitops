## Top-3 (для spec-writer)

### 1. go-upgrade-1-26: Обновление Go toolchain до 1.26.2
**Категория:** security
**Impact:** 5
**Effort:** 2
**Обоснование:** Go 1.26.2 закрывает security-уязвимости в crypto/tls и crypto/x509 — пакетах, которые mctl-agent использует при каждом TLS-соединении с GitHub API и Anthropic API. Текущий toolchain 1.24 отстаёт на два minor-релиза, что означает накопленный долг по CVE в стандартной библиотеке. Апгрейд бесплатно включает Green Tea GC (10–40% снижение GC overhead) без роста потребления памяти, что безопасно даже с учётом ограничений тенанта labs.
**Source:** https://go.dev/doc/devel/release — golang/go go1.26.2 (2026-04-07); https://go.dev/blog/go1.26 — Green Tea GC

### 2. argo-workflows-cve-patch: Обновление Argo Workflows до v4.0.5 (CVE-2026-40886)
**Категория:** security
**Impact:** 4
**Effort:** 2
**Обоснование:** CVE-2026-40886 (High 7.7) позволяет вызвать panic Argo Workflows controller через malformed pod annotation, что полностью останавливает обработку всех Workflow в кластере. mctl-agent критически зависит от CronWorkflow `cwft-rotate-github-token`: при падении контроллера ротация GitHub App installation token прекращается, и через 30 минут mctl-agent теряет доступ к GitHub — PR-creation (основная функция self-healing) перестаёт работать. Fix уже выпущен в v4.0.5 от 2026-04-23.
**Source:** https://advisories.gitlab.com/golang/github.com/argoproj/argo-workflows/v4/CVE-2026-40886/ — CVE-2026-40886; https://github.com/argoproj/argo-workflows/releases/latest — argo-workflows v4.0.5

### 3. go-github-v85-authz-fix: Обновление google/go-github до v85 (Authorization header leak)
**Категория:** security
**Impact:** 4
**Effort:** 3
**Обоснование:** В go-github v85 добавлен cross-host redirect rejection: ранее при HTTP-редиректе на другой хост заголовок Authorization с GitHub App installation token передавался стороннему серверу — это прямая утечка секрета, дающего права на запись в mctl-gitops. mctl-agent использует v68, разрыв составляет 17 мажорных версий; breaking changes (Custom Org Role API, MarkThreadDone signature) требуют небольшой правки кода, но security-ценность однозначно оправдывает Effort.
**Source:** https://github.com/google/go-github/releases/latest — google/go-github v85.0.0 cross-host redirect rejection

## Отброшено

- **chi v5.2.5 / CVE-2025-69725**: mctl-agent использует chi 5.2.1, CVE затрагивает только >=5.2.2 — угрозы нет; апгрейд chi без security-мотивации не приоритетен.
- **anthropic-sdk-go v1.38.0 (CMA Memory, structured outputs)**: feature/DX, не security; nice-to-have для будущего proposal по расширению LLMDiagnosis.
- **modernc.org/sqlite v1.50.0**: специфичных CVE не обнаружено, 16-версионный апгрейд без security-мотивации — низкий приоритет.
- **alertmanager v0.32.0**: mctl-agent является получателем webhook, а не оператором alertmanager; upstream-изменения templating не ломают текущий webhook contract для mctl-agent.
- **ArgoCD Image Updater CVE-2026-6388 (Critical 9.1)**: затрагивает отдельный компонент ArgoCD Image Updater, не сам ArgoCD и не mctl-agent — вне зоны ответственности сервиса.
- **CVE-2026-31892 / CVE-2026-28229 (Argo Workflows)**: релевантны платформе, но fix для них уже включён в v4.0.5 (покрывается находкой #2 — argo-workflows-cve-patch).
- **argo-cd v3.3.8**: только bug fixes в application controller, CRD-изменений нет; mctl-agent не зависит от ArgoCD API напрямую.
