## Top-3 (for spec-writer)

### 1. go-upgrade-1-26: Upgrade Go toolchain to 1.26.2
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Go 1.26.2 closes security vulnerabilities in crypto/tls and crypto/x509 — packages that mctl-agent uses on every TLS connection to GitHub API and Anthropic API. The current toolchain 1.24 is two minor releases behind, which means accumulated CVE debt in the standard library. The upgrade transparently includes Green Tea GC (10–40% reduction in GC overhead) without an increase in memory consumption, which is safe even given the labs tenant constraints.
**Source:** https://go.dev/doc/devel/release — golang/go go1.26.2 (2026-04-07); https://go.dev/blog/go1.26 — Green Tea GC

### 2. argo-workflows-cve-patch: Upgrade Argo Workflows to v4.0.5 (CVE-2026-40886)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-40886 (High 7.7) lets a malformed pod annotation cause the Argo Workflows controller to panic, which fully halts processing of all Workflows in the cluster. mctl-agent critically depends on the CronWorkflow `cwft-rotate-github-token`: when the controller falls over, GitHub App installation token rotation stops, and within 30 minutes mctl-agent loses access to GitHub — PR creation (the core self-healing function) stops working. The fix is already shipped in v4.0.5 dated 2026-04-23.
**Source:** https://advisories.gitlab.com/golang/github.com/argoproj/argo-workflows/v4/CVE-2026-40886/ — CVE-2026-40886; https://github.com/argoproj/argo-workflows/releases/latest — argo-workflows v4.0.5

### 3. go-github-v85-authz-fix: Upgrade google/go-github to v85 (Authorization header leak)
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** go-github v85 adds cross-host redirect rejection: previously, on an HTTP redirect to a different host, the Authorization header carrying the GitHub App installation token was passed to the third-party server — a direct leak of a secret with write access to mctl-gitops. mctl-agent uses v68, a gap of 17 major versions; breaking changes (Custom Org Role API, MarkThreadDone signature) require a small code change, but the security value clearly justifies the Effort.
**Source:** https://github.com/google/go-github/releases/latest — google/go-github v85.0.0 cross-host redirect rejection
