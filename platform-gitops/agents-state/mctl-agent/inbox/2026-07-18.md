Current pin (5.2.1) predates the vulnerable range per this advisory's stated "affected >=5.2.2" — needs analyst to double check applicability. https://github.com/advisories/GHSA-4h76-926q-wxxw
- **Go toolchain security releases, Jul 2026** — go1.26.4/1.25.11 fixed CVE-2026-42504 (quadratic-cost MIME `WordDecoder.DecodeHeader` on crafted headers → CPU exhaustion), a `net/textproto` error-sanitization issue, and a `crypto/x509` quadratic hostname-verification cost issue (CVE-2026-27145 referenced). go1.26.5/1.25.12 (Jul 7 2026) then fixed an `os.Root` symlink-escape bug and a `crypto/tls` Encrypted Client Hello privacy leak; Red Hat's Jul 9 2026 advisory bundled CVE-2026-39821 (`golang.org/x/net/idna` Punycode privilege escalation) and CVE-2026-39822 (`os.Root` symlink following). mctl-agent is pinned to go1.24, which is outside Go's two-latest-major security-support window, so none of these backports land on it. https://groups.google.com/g/golang-announce/c/qcCIEXso47M and https://access.redhat.com/errata/RHSA-2026:37435
