## Top-3 (for spec-writer)

### 1. go-runtime-upgrade: Upgrade Go runtime from 1.24 to 1.26.2
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Three CVEs (CVE-2026-32283, CVE-2026-32280, CVE-2026-32281) in `crypto/tls` and `crypto/x509` enable denial-of-service attacks against any Go 1.24 service that handles TLS connections or verifies certificates. mctl-agent receives HTTPS webhooks (AlertManager, Telegram) and makes outbound HTTPS calls (GitHub API, Anthropic API), making it directly exposed. Go 1.26.2 also ships the Green Tea GC (10–40 % GC overhead reduction), which benefits a persistent service under alert load. Upgrade is a module-level go.mod change with low breakage risk between 1.24 and 1.26.
**Source:** CVE-2026-32283/32280/32281 — https://github.com/golang/go/issues/78334; Go 1.26.2 release — https://github.com/golang/go/releases/latest

### 2. chi-open-redirect: Upgrade go-chi/chi to v5.2.5 (open-redirect CVE)
**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** GHSA-mqqf-5wvp-8fh8 documents an open-redirect vulnerability in chi's `RedirectSlashes` middleware. Although the advisory's confirmed-vulnerable range starts at v5.2.2 (current is v5.2.1), the root cause — insufficient URL validation — is structurally present in older versions too, and the fix landed in v5.2.4. mctl-agent's AlertManager and Telegram webhook endpoints could be abused in a phishing redirect if the middleware is active. Upgrading to v5.2.5 (latest) eliminates the risk with minimal effort — only a go.mod version bump.
**Source:** GHSA-mqqf-5wvp-8fh8 — https://pkg.go.dev/vuln/GO-2026-4316; chi v5.2.5 release — https://github.com/go-chi/chi/releases/tag/v5.2.5

### 3. sqlite-upgrade: Upgrade modernc.org/sqlite from v1.34 to v1.50.0 (CVE + API improvements)
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** CVE-2025-70873 (CVSS 7.5 High) is a heap information disclosure in SQLite's zipfile extension, which `modernc.org/sqlite` inherits through code transpilation. While mctl-agent's tickets-DB and skill-metrics usage does not load ZIP files, the flaw exists in the binary and could be triggered if any future code path reads a crafted file. The v1.50.0 upgrade also adds the `ColumnInfo` API (useful for future schema introspection tooling) and the jump from v1.34 covers 16 minor releases of upstream fixes. Upgrade is a drop-in Go module bump with no breaking API changes.
**Source:** CVE-2025-70873 — https://sqlite.org/cves.html; modernc.org/sqlite v1.50.0 — https://pkg.go.dev/modernc.org/sqlite
