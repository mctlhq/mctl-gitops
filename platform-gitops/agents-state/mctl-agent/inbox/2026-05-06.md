## Top-3 (for spec-writer)

### 1. mcp-csrf-origin-check: MCP endpoint CSRF protection via Origin header validation
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-33252 directly targets the Streamable HTTP MCP transport used by mctl-agent's `POST /mcp` endpoint, allowing arbitrary MCP tool execution from a malicious website without Origin header validation or Content-Type enforcement. The six MCP tools exposed can trigger diagnosis, ticket mutations, and skill registration — making this a high-blast-radius attack surface. The fix is a focused middleware addition on the `/mcp` route in chi and does not affect other endpoints.
**Source:** CVE-2026-33252 — https://osv.dev (reported via GitHub Advisory Database)

### 2. go-runtime-upgrade: Upgrade Go runtime from 1.24 to 1.26.2
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Three stdlib CVEs (CVE-2026-32283 crypto/tls TLS 1.3 deadlock DoS, CVE-2026-32280 crypto/x509 chain-building DoS, CVE-2026-32289 html/template XSS) are fixed only in Go 1.25.9 or 1.26.2; Go 1.24 will not receive these patches. mctl-agent makes outbound TLS calls to the Anthropic API and GitHub App endpoints, putting both crypto/tls and crypto/x509 paths in active use. Upgrading to 1.26.2 is the only remediation path and also picks up the chi v5.2.5 Go-1.22+ minimum without a separate chi bump.
**Source:** CVE-2026-32283, CVE-2026-32280, CVE-2026-32289 — https://pkg.go.dev/vuln/list; golang/go go1.26.2 release — https://go.dev/doc/devel/release

### 3. sqlite-zipfile-cve-audit: Audit and patch modernc.org/sqlite for CVE-2025-70873
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** CVE-2025-70873 is a heap info-disclosure in SQLite's zipfile extension affecting v3.51.1 and earlier; modernc.org/sqlite 1.34 vendors upstream SQLite and the exact bundled version must be verified. mctl-agent uses SQLite for the tickets DB and skill metrics, so if the vulnerable extension is compiled in, any process or alert payload that triggers a ZIP-processing code path could leak heap memory. The fix is a `go get modernc.org/sqlite@latest` bump once the safe version is confirmed, which carries no schema or API changes.
**Source:** CVE-2025-70873 — https://sqlite.org/cves.html; modernc.org/sqlite 1.34 in service stack
