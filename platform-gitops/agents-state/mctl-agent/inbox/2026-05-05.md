## Top-3 (for spec-writer)

### 1. go-github-v85-upgrade: Upgrade google/go-github v68 → v85 (Authorization header leak fix)
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** mctl-agent uses go-github as its sole GitHub integration layer for creating fix PRs. v85 ships a direct security fix that rejects cross-host HTTP redirects, preventing the `Authorization` header (GitHub installation token) from leaking to a third-party host. The current v68 is 17 major versions behind. The token is rotated every 30 minutes (CronWorkflow `cwft-rotate-github-token`) but a leak window still exists. Three breaking API signature changes must be handled: `GetOrgRole`, `CreateCustomOrgRole`, `UpdateCustomOrgRole`, and `MarkThreadDone`.
**Source:** https://github.com/google/go-github/releases/tag/v85.0.0 — security: reject cross-host redirects.

### 2. anthropic-sdk-timeout-default: Upgrade anthropic-sdk-go and enforce diagnose-phase timeout
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** The LLMDiagnosis skill (the fallback for all unmatched alerts) calls the Anthropic API with no documented HTTP timeout in the current codebase. v1.39.0 of the SDK ships a default 10-minute client timeout. Without a hard deadline the diagnose goroutine can block indefinitely, preventing ticket closure and starving the skill pipeline. Upgrading and explicitly configuring a timeout (≤ 5 minutes recommended) closes this reliability gap; the effort is low because no public API changes were made.
**Source:** https://github.com/anthropics/anthropic-sdk-go/releases/tag/v1.39.0 — "Added 10-minute timeout to client requests".

### 3. sqlite-v150-upgrade: Upgrade modernc.org/sqlite v1.34 → v1.50.0 (CVE-2025-6965 remediation)
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** CVE-2025-6965 is a buffer overflow in SQLite < 3.50.2. `modernc.org/sqlite` v1.34 (released early 2025) bundles an SQLite version older than 3.50.2 and is therefore likely affected. The tickets DB and skill-metrics store are both backed by this library; a crafted query could corrupt in-process memory. Upgrading to v1.50.0 (April 2026, bundles SQLite ≥ 3.50.x) resolves the CVE with no schema migration required; the `modernc.org/sqlite` public Go API is backward-compatible.
**Source:** CVE-2025-6965 — SQLite buffer overflow < 3.50.2; modernc.org/sqlite v1.50.0 released 2026-04-24.
