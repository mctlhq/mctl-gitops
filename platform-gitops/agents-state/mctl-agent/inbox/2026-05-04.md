## Source: cve

- **CVE-2026-40886 (Argo Workflows, CVSS 7.7 High)** — Unchecked array index in `podGCFromPod()` causes controller-wide panic when a workflow pod carries a malformed `workflows.argoproj.io/pod-gc-strategy` annotation; any user with `create` on Workflow resources can trigger permanent DoS. Fixed in v3.7.14 and v4.0.5. https://dailycve.com/argo-workflows-unchecked-array-index-cve-2026-40886-critical/

- **CVE-2026-31892 (Argo Workflows, security bypass)** — Users who can submit Workflows can bypass all security policies set in a `WorkflowTemplate` by injecting a `podSpecPatch` field, even when `templateReferencing: Strict` is configured. - **CVE-2026-32280 / CVE-2026-32281 (Go crypto/x509, DoS)** — Two denial-of-service vulnerabilities published 2026-04-07 in `crypto/x509` chain-building: excessive work with large intermediate certificate sets and inefficient policy-mapping validation. ## Top-3 (for spec-writer)

### 1. bump-go-github: Upgrade google/go-github v68 to v85 (Authorization header leak)
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** mctl-agent directly calls go-github to create fix PRs using a short-lived GitHub App installation token; v85 blocks cross-host redirects that would silently forward that token to an attacker-controlled host. The gap of 17 major versions also means the client's URL path validation (blocking `..` segments) and webhook payload size limits are absent. Closing this closes a concrete credential-leak vector on the PR creation path.
**Source:** https://github.com/google/go-github/releases/latest (v85.0.0 release notes, no standalone CVE)

### 2. bump-go-toolchain: Upgrade Go 1.24 to 1.26.2 (CVE-2026-32280 / CVE-2026-32281 crypto/x509 DoS)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** mctl-agent makes continuous outbound TLS connections to the GitHub API and Anthropic API; the two crypto/x509 chain-building DoS CVEs mean a crafted TLS certificate presented by a compromised intermediate CA could exhaust CPU and halt the agent entirely. Upgrading the Go toolchain is a self-contained build change with no API surface impact, and it bundles fixes in crypto/tls, archive/tar, and net/http as well.
**Source:** https://pkg.go.dev/vuln/list (CVE-2026-32280, CVE-2026-32281, fixed in Go 1.26.2)

### 3. bump-chi: Upgrade go-chi/chi v5.2.1 to v5.2.5 (CVE-2025-69725 open redirect)
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** mctl-agent exposes chi-routed endpoints publicly at https://agent.mctl.ai; although the open-redirect CVE technically requires RedirectSlashes middleware to be active, upgrading to v5.2.5 is a single-line go.mod change that also eliminates the RouteHeaders double-invocation bug and aligns with the new Go 1.22 minimum already required by the updated toolchain. The effort is minimal and the patch is already available.
**Source:** https://github.com/go-chi/chi/security/advisories/GHSA-mqqf-5wvp-8fh8 (CVE-2025-69725, CVSS 4.7)

## Dropped
- **ArgoCD GHSA-3v3m-wc6v-x4x3 (CVSS 9.6):** mctl-agent does not vendor ArgoCD Go libraries; this is a platform ops concern (upgrade the ArgoCD deployment). No mctl-agent code change required.
- **Argo Workflows CVE-2026-40886 / CVE-2026-31892:** Same reasoning — mctl-agent receives Argo webhook payloads but does not import Argo Workflows libraries. Remediation is a cluster-level upgrade by the ops team.

- **anthropic-sdk-go v1.38.0:** No security fixes; new features (CMA Memory public beta, structured outputs) are nice-to-have and do not meet the security/performance bar for this cycle.
- **modernc.org/sqlite v1.50.0:** No security fixes listed; the ColumnInfo struct addition is a minor DX improvement with no impact on the current schema or queries.
- **prometheus/alertmanager v0.32.1:** No webhook contract changes and no security fixes; mctl-agent is a consumer of AlertManager webhooks, not a maintainer of AlertManager itself.
