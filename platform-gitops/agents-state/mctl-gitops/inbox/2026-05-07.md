## Top-3 (for spec-writer)

### 1. vault-v2-major-upgrade-plan: Vault v2.0.0 Major Version Upgrade and CVE-2026-5807 Remediation
**Category:** security
**Impact:** 5
**Effort:** 4
**Rationale:** Vault v2.0.0 fixes four CVEs; CVE-2026-5807 (unauthenticated DoS via root-token generation, no existing proposal) is included. Individual CVE patches exist but no migration plan covers the v2.0.0 breaking changes (path canonicalization, Docker helpers migration, UBI 10). Without a coordinated upgrade plan, piecemeal patching risks misconfiguration. Vault underpins all secrets management via the `vault-backend` ClusterSecretStore.
**Source:** https://github.com/hashicorp/vault/releases/latest ; https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-5807/

### 2. cert-manager-dns-dos-patch: cert-manager DoS via Crafted DNS Response (GHSA-gx3x-vq4p-mhhv)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** No existing cert-manager proposal exists. GHSA-gx3x-vq4p-mhhv allows a crafted DNS response to panic the cert-manager controller, halting certificate issuance and renewal platform-wide; this would take down TLS for all services at renewal time. Fixed in v1.20.2 (already released April 2026), which also includes a Go 1.26.2 security bump and a webhook YAML fix.
**Source:** https://github.com/cert-manager/cert-manager/security ; https://github.com/cert-manager/cert-manager/releases/latest

### 3. argocd-v3-4-1-cluster-gen-migration: ArgoCD v3.4.1 Cluster-Generator Label Format Migration
**Category:** feature
**Impact:** 4
**Effort:** 3
**Rationale:** ArgoCD v3.4.1 (released today) introduces a breaking change: cluster generators must now use `argocd.argoproj.io/kubernetes-version` in `vMajor.Minor.Patch` format aligned with Helm 3.19.0. Both existing ArgoCD 3.4 upgrade plans targeted v3.4.0 and do not include this migration task. ApplicationSets using the old label format will fail silently or error out after upgrade, potentially leaving services out of sync.
**Source:** https://github.com/argoproj/argo-cd/releases/latest
