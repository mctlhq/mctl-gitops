- **CVE-2026-35469 (argoproj/argo-rollouts)** — Fixed in v1.9.1 (released 2026-07-17); NVD detail page restricted/inaccessible at fetch time, fix was cherry-picked from PR #4814 onto the 1.9 branch — severity/details need follow-up verification. https://github.com/argoproj/argo-rollouts/releases

- **GHSA-8rvj-mm4h-c258 (cert-manager/cert-manager)** — High severity: default `cert-manager-edit` aggregate ClusterRole let namespace users create ACME Challenge/Order resources directly, bypassing Issuer DNS01 solver policy and exfiltrating ClusterIssuer DNS credentials (notably via acme-dns provider); affects >=1.18.0, <=1.20.2, fixed in v1.19.6 and v1.20.3. https://github.com/cert-manager/cert-manager/security/advisories/GHSA-8rvj-mm4h-c258
- **CVE-2026-5052 / HCSEC-2026-06 (hashicorp/vault)** — SSRF in Vault PKI's ACME validation logic: does not reject local-network targets for http-01/tls-alpn-01 challenges, allowing an attacker who controls DNS for an owned domain to make Vault fetch an internal target. https://www.sentinelone.com/vulnerability-database/cve-2026-5052/
- **CVE-2026-3605 / HCSEC-2026-05 (hashicorp/vault)** — KVv2 secrets engine bug: an authenticated user with a glob-pattern policy on a KVv2 path can delete secrets outside their authorized read/write scope; fixed in Vault CE/Enterprise 2.0.0, and backported to Enterprise 1.21.5 / 1.20.10 / 1.19.16. https://www.sentinelone.com/vulnerability-database/cve-2026-3605/
