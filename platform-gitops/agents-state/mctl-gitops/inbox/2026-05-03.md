## Top-3 (for spec-writer)

### 1. argocd-xss-version-verify: Verify ArgoCD deployment is not exposed to CVE-2025-47933 (Critical XSS)
**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** CVE-2025-47933 is a Critical CVSS 9.0 XSS that affects ArgoCD v1.2.0-rc1 through v3.0.3; the current deployed version v3.3.9 is outside the affected range and therefore patched. However, no existing proposal documents this verification, and there is no audit record confirming the cluster is not running an older vulnerable version. A lightweight proposal to query the running ArgoCD version, record the confirmation, and add a version-floor check to the bootstrap chart prevents regression if someone accidentally pins back to an older image.
**Source:** https://github.com/advisories/GHSA-2hj5-g64g-fp6p — CVE-2025-47933 (argo-cd, Critical CVSS 9.0) in today's inbox.

### 2. prometheus-security-patch: Pin and verify Prometheus v3.11.3 to remediate credential exposure and XSS
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** Prometheus v3.11.3 patched three security issues: AzureAD OAuth credential exposure (credentials leaked in debug output), snappy decompression DoS, and stored XSS in the UI heatmap chart. Prometheus is an explicit dependency in the architecture and templated through this repo. No existing proposal covers any of these Prometheus CVEs, so a targeted version-pin and values-file audit is needed to confirm v3.11.3 is deployed across both tenants. No memory impact is expected in `labs` since this is a patch-level update.
**Source:** https://github.com/prometheus/prometheus/releases/latest — Prometheus v3.11.3 release notes in today's inbox.

### 3. argocd-v3-4-upgrade-plan: Prepare upgrade path for ArgoCD v3.4.0 once GA
**Category:** feature
**Impact:** 2
**Effort:** 2
**Rationale:** ArgoCD v3.4.0 RC7 was published April 30, 2026, indicating GA is close. Preparing a structured upgrade plan now — changelog review, ApplicationSet API diff, staging rollout in `labs` before `admins` — avoids scrambling at GA release. This is a minor version increment (v3.3 to v3.4) so it does not violate the constraint against major upgrades on patch day. The `labs` staging rollout should be assessed for memory impact before promotion.
**Source:** https://github.com/argoproj/argo-cd/releases/latest — argo-cd v3.4.0 RC7 (Apr 30, 2026) in today's inbox.
