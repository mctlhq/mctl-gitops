## Top-3 (for spec-writer)

### 1. argo-workflows-cve-patch: Patch three CVEs in Argo Workflows up to v4.0.5

**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Three CVEs in a single component require a single upgrade: CVE-2026-40886 (CVSS 7.7) crashes the entire controller via a malformed annotation — any tenant can take down all CronWorkflow/ClusterWorkflowTemplate executions in the cluster; CVE-2026-28229 exposes WorkflowTemplate contents (including embedded secrets) over a Bearer token; CVE-2026-31892 lets a user bypass WorkflowTemplate security settings via podSpecPatch. All three are closed in v4.0.5, which makes the upgrade the single action with the maximum security ROI.
**Source:** https://github.com/argoproj/argo-workflows/releases/tag/v4.0.5 ; CVE-2026-40886, CVE-2026-28229, CVE-2026-31892

### 2. helm-security-patch: Upgrade Helm to v4.1.4 (three GHSAs, including path traversal and arbitrary file write)

**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** Three Helm vulnerabilities are closed by a single patch release v4.1.4: GHSA-vmx8-mqv2-9gmg permits arbitrary file writes outside the plugin directory, GHSA-hr2v-4r36-88hr enables path traversal during chart extraction, GHSA-q5jf-9vfq-h4h7 bypasses plugin signature verification. Helm is used in all deployments via the base-service chart and the Helm-based ApplicationSet, so vulnerabilities in the CLI and chart extraction directly affect the platform. Effort is minimal: a patch release without breaking changes.
**Source:** https://github.com/helm/helm/releases/tag/v4.1.4 ; GHSA-hr2v-4r36-88hr, GHSA-q5jf-9vfq-h4h7, GHSA-vmx8-mqv2-9gmg

### 3. eso-cve-patch: Upgrade External Secrets Operator to v2.4.0 (CVE-2026-34165, CVE-2026-33762)

**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** ESO is the single delivery channel for all tenant secrets from Vault, so any vulnerability in it has a high blast radius. Two CVEs (CVE-2026-34165, CVE-2026-33762) are closed in v2.4.0; details are not disclosed, which suggests a possible embargoed publication with an intentionally limited description. v2.4.0 also improves the VaultDynamicSecret GET method, which can be useful for the platform's vault-backend ClusterSecretStore. Memory consumption does not increase — safe for the labs tenant.
**Source:** https://github.com/external-secrets/external-secrets/releases/tag/v2.4.0 ; CVE-2026-34165, CVE-2026-33762
