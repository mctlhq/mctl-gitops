## Top-3 (для spec-writer)

### 1. argo-workflows-cve-patch: Патч трёх CVE в Argo Workflows до v4.0.5

**Категория:** security
**Impact:** 5
**Effort:** 2
**Обоснование:** Три CVE в одном компоненте требуют единого апгрейда: CVE-2026-40886 (CVSS 7.7) вызывает crash loop всего контроллера через malformed annotation — любой тенант может уронить все CronWorkflow/ClusterWorkflowTemplate кластера; CVE-2026-28229 открывает содержимое WorkflowTemplate (включая embedded secrets) по Bearer token; CVE-2026-31892 позволяет обойти security settings WorkflowTemplate через podSpecPatch. Все три закрыты в v4.0.5, что делает апгрейд единственным действием с максимальным security ROI.
**Source:** https://github.com/argoproj/argo-workflows/releases/tag/v4.0.5 ; CVE-2026-40886, CVE-2026-28229, CVE-2026-31892

### 2. helm-security-patch: Апгрейд Helm до v4.1.4 (три GHSA, включая path traversal и arbitrary file write)

**Категория:** security
**Impact:** 4
**Effort:** 1
**Обоснование:** Три уязвимости в Helm закрываются одним patch-релизом v4.1.4: GHSA-vmx8-mqv2-9gmg допускает произвольную запись файлов за пределами директории плагина, GHSA-hr2v-4r36-88hr позволяет path traversal при распаковке chart, GHSA-q5jf-9vfq-h4h7 обходит проверку подписей плагинов. Helm используется во всех деплоях через base-service chart и Helm-based ApplicationSet, поэтому уязвимость в CLI и chart extraction непосредственно затрагивает платформу. Effort минимальный: patch-релиз без breaking changes.
**Source:** https://github.com/helm/helm/releases/tag/v4.1.4 ; GHSA-hr2v-4r36-88hr, GHSA-q5jf-9vfq-h4h7, GHSA-vmx8-mqv2-9gmg

### 3. eso-cve-patch: Апгрейд External Secrets Operator до v2.4.0 (CVE-2026-34165, CVE-2026-33762)

**Категория:** security
**Impact:** 4
**Effort:** 2
**Обоснование:** ESO — единственный канал доставки секретов всех тенантов из Vault, поэтому любая уязвимость в нём имеет высокий blast radius. Два CVE (CVE-2026-34165, CVE-2026-33762) закрыты в v2.4.0, детали не раскрыты, что указывает на возможную embargo-публикацию с намеренно ограниченным описанием. v2.4.0 также улучшает VaultDynamicSecret GET method, что может быть полезно для vault-backend ClusterSecretStore платформы. Не увеличивает потребление памяти — безопасно для тенанта labs.
**Source:** https://github.com/external-secrets/external-secrets/releases/tag/v2.4.0 ; CVE-2026-34165, CVE-2026-33762
