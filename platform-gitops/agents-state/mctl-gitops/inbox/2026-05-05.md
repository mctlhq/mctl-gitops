- **CVE-2026-31892 (Argo Workflows, High 8.9)** — podSpecPatch field on a submitted Workflow bypasses templateReferencing: Strict mode, enabling container escape and potential node compromise; affects 2.9.0–3.7.10 and 4.0.0–4.0.1, fixed in 3.7.11 and 4.0.2. https://advisories.gitlab.com/pkg/golang/github.com/argoproj/argo-workflows/CVE-2026-31892/

- **CVE-2026-28229 (Argo Workflows)** — WorkflowTemplate and ClusterWorkflowTemplate endpoints accessible with any bearer token including invalid ones, leaking sensitive template content; fixed in 4.0.2. https://advisories.gitlab.com/pkg/golang/github.com/argoproj/argo-workflows/v4/CVE-2026-28229/

## Top-3 (for spec-writer)

### 1. argocd-secret-leakage-patch: Upgrade ArgoCD to v3.3.9 to fix Critical Secret Leakage (CVE-2026-42880)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-42880 scores 9.6 Critical and allows any authenticated user to extract plaintext Kubernetes Secret values through the ServerSideDiff endpoint. ArgoCD is the sync engine for the entire platform (ops.mctl.ai) and has access to all tenant secrets, making this the highest-priority finding. A patch release (v3.3.9) is already available, keeping upgrade effort low.
**Source:** https://github.com/argoproj/argo-cd/security/advisories/GHSA-3v3m-wc6v-x4x3 — CVE-2026-42880, ArgoCD v3.3.9 release notes.

### 2. argo-workflows-cve-bundle-upgrade: Upgrade Argo Workflows to v3.7.14 to fix CVE-2026-31892 and CVE-2026-40886
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-31892 (High 8.9) allows a submitted Workflow to bypass templateReferencing: Strict mode via podSpecPatch, enabling container escape and potential node compromise — this directly targets our CronWorkflow and ClusterWorkflowTemplate setup. CVE-2026-28229 additionally allows unauthenticated access to ClusterWorkflowTemplate content, leaking sensitive pipeline definitions. Both are fixed in v3.7.14, which also resolves the pre-existing v3.7.10 archive INSERT NULL bug documented in architecture.md, making the upgrade triply beneficial. Note: the archive bug itself is already a known issue and is not being re-proposed here; this finding is the security patch bundle.
**Source:** https://advisories.gitlab.com/pkg/golang/github.com/argoproj/argo-workflows/CVE-2026-31892/ — CVE-2026-31892 (High 8.9); https://github.com/argoproj/argo-workflows/releases/tag/v3.7.14

### 3. eso-cross-namespace-bypass-patch: Upgrade External Secrets Operator to remediate CVE-2026-22822 Cross-Namespace Secret Access
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-22822 (Critical 8.8) allows any user who can create or update an ExternalSecret resource to read secrets from any namespace using the ESO controller's ClusterRoleBinding, entirely defeating namespace isolation. We rely on ESO + Vault for all secrets across both the admins and labs tenants, so exploitation would expose the full secret surface of the platform. The ESO helm-chart-2.4.1 (April 28, 2026) is the current release and is expected to include the remediation; confirmation of the exact fixed chart version should be validated during implementation.
**Source:** https://vulert.com/vuln-db/CVE-2026-22822 — CVE-2026-22822; https://github.com/external-secrets/external-secrets/releases/tag/helm-chart-2.4.1

## Dropped
- **CVE-2026-35204 / Helm v4.1.4** — Path traversal affects plugin.yaml in Helm plugins. We use Helm for chart packaging and rendering, not for distributing or installing third-party plugins in CI, so the attack surface is limited. Actionable in a follow-up cycle once the security-critical items above are resolved.
- **CVE-2026-5807 / CVE-2026-4525 / CVE-2026-5052 / CVE-2026-3605 (Vault CVEs)** — All four are real but the latest Vault release is v2.0.0, a major version with breaking changes (double-slash path rejection, root generation auth requirements, renamed UI paths). Upgrading Vault server on patch-day alongside other critical upgrades is explicitly called out in architecture.md as risky. A dedicated Vault upgrade proposal with migration planning is the appropriate track.
- **CVE-2026-34984 (ESO Sprig getHostByName)** — Affects ESO versions 2.2.0 and below; almost certainly already resolved in our current ESO deployment. No action required unless version audit confirms otherwise.
- **CVE-2026-21726 (Grafana Loki path traversal)** — Loki is not listed as a core dependency in architecture.md or current-version.md; not confirmed to be in our managed stack, so dropped per architectural-fit criterion.
- **CVE-2026-42151 / CVE-2026-42154 (Prometheus)** — Prometheus is referenced only as a potential templated dependency. AzureAD OAuth exposure and snappy decode limit bypass are real but Prometheus is not confirmed as a managed component in mctl-gitops scope.
- **Argo Rollouts v1.9.0** — No CVEs listed; changes are operational fixes. Not a security priority this cycle.
- **cert-manager v1.20.2** — Go bump and YAML generation fix; low urgency relative to the Critical CVEs above.
- **Vault v2.0.0 major upgrade** — Breaking changes require a dedicated migration plan; out of scope for a single patch-day proposal.
