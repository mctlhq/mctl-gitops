## Source: cve

- **CVE-2025-55190 (Critical, CVSS 10.0) — Argo CD Project API Token Exposes Repository Credentials** — API tokens with project-level get permissions can retrieve all repository credentials (usernames, passwords) via the `/api/v1/projects/{project}/detailed` endpoint regardless of explicit secret-access grants; affects v2.13.0–2.13.8, v2.14.0–2.14.15, v3.0.0–v3.0.12, v3.1.0-rc1–v3.1.1; fixed in v3.1.2, v3.0.14, v2.14.16, v2.13.9. https://nvd.nist.gov/vuln/detail/CVE-2025-55190

- **CVE-2026-4525 (High, CVSS 7.5) — HashiCorp Vault Token Exposure via Incorrect Header Sanitization** — When a Vault auth mount is configured to pass through the `Authorization` header, Vault incorrectly forwards the caller's Vault token to the backend auth plugin, potentially exposing the token to the plugin; affects v0.11.2–v1.21.4, fixed in v2.0.0 (Community) and v1.21.5, v1.20.10, v1.19.16 (Enterprise). https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-4525/

- **CVE-2026-5807 (High, CVSS 7.5) — HashiCorp Vault DoS via Unauthenticated Root Token Generation/Rekey Operations** — An unauthenticated attacker can repeatedly initiate or cancel root token generation or rekey operations on `sys/rekey`, `sys/generate-root`, and `sys/rekey-recovery-key` endpoints, monopolizing the single in-progress operation slot and blocking legitimate administrative workflows; affects all Vault versions up to v1.21.4, fixed in v2.0.0 and Enterprise v1.21.5, v1.20.10, v1.19.16. https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-5807/

- **CVE-2026-34984 (High) — External Secrets Operator DNS-based Secret Exfiltration via getHostByName in v2 Template Engine** — The v2 template engine in ESO leaves `getHostByName` accessible in user-controlled templates; an attacker who can create or update a templated ExternalSecret can trigger controller-side DNS lookups encoding secret values in queries, exfiltrating secret material without direct outbound network access; affects ESO v2.0.0–v2.2.0 and pre-release builds before 6800989bdc12. https://advisories.gitlab.com/pkg/golang/github.com/external-secrets/external-secrets/CVE-2026-34984

## Top-3 (for spec-writer)

### 1. argocd-repo-cred-leak-patch: ArgoCD CVE-2025-55190 — Project API Token Exposes Repository Credentials
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVSS 10.0 critical; any project-scoped API token holder can retrieve all repository credentials (GitHub deploy key password, bot credentials) via the `/api/v1/projects/{project}/detailed` endpoint with no additional privilege required. ArgoCD is the central sync engine for the entire platform — credential leakage here compromises every tenant's workloads. The fix is a point upgrade to v3.0.14+ or v3.1.2+; v3.3.9 (released April 30, 2026) is confirmed safe and also ships ApplicationSet generator panic fixes, making it the target version.
**Source:** https://nvd.nist.gov/vuln/detail/CVE-2025-55190

### 2. eso-dns-exfil-patch: External Secrets Operator CVE-2026-34984 — DNS-based Secret Exfiltration via v2 Template Engine
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** ESO is deployed as the ClusterSecretStore `vault-backend` bridging Vault to all tenant namespaces; ExternalSecret manifests are checked in under `platform-gitops/argo-workflows/secrets/`. The `getHostByName` function left accessible in user-controlled templates allows covert secret exfiltration purely via DNS queries, bypassing egress firewall rules. The fix requires upgrading ESO past commit 6800989bdc12 (helm-chart-2.4.1 is already released and likely contains the patch, but must be verified). This is distinct from the previously proposed eso-cve-patch.
**Source:** https://advisories.gitlab.com/pkg/golang/github.com/external-secrets/external-secrets/CVE-2026-34984

### 3. vault-cve-2026-token-exposure: HashiCorp Vault CVE-2026-4525 + CVE-2026-5807 — Token Exposure and Administrative DoS
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** Vault is the platform's central secret store (`secrets.mctl.ai`) backing ESO for all tenants. CVE-2026-4525 leaks caller Vault tokens to backend auth plugins via incorrect header sanitization, risking privilege escalation across auth mounts. CVE-2026-5807 allows an unauthenticated attacker to monopolize the rekey/root-token-generation operation slot, blocking all administrative recovery workflows. Both are fixed in v2.0.0 (Community), which is already released. These are distinct CVEs from the previously proposed vault-ssrf-acme-patch and vault-kvv2-deletion-bypass proposals. Upgrading Vault is a 3-effort task given the major version bump and required migration testing.
**Source:** https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-4525/ ; https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-5807/
