## Top-3 (for spec-writer)

### 1. argocd-v3-4-upgrade-plan: Execute ArgoCD v3.4.0 Upgrade (Now GA)
**Category:** feature
**Impact:** 4
**Effort:** 2
**Rationale:** ArgoCD v3.4.0 reached GA on May 5, 2026, and the existing `argocd-v3-4-upgrade-plan` proposal explicitly states it is a preparation plan only, requiring a separate sign-off after labs stability is confirmed. That gate condition is now met — the GA release is available and the preparation work is done, making an execution spec the direct next step. Upgrading to v3.4.x also consolidates the platform on a version that includes all v3.3.x security patches, reducing the window of exposure for any future v3.3-branch CVEs.
**Source:** https://github.com/argoproj/argo-cd/releases/tag/v3.4.0

### 2. grafana-dos-patch: Remediate Grafana Memory Exhaustion DoS (CVE-2026-27880)
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** CVE-2026-27880 (CVSS 7.5) is a distinct attack vector from the RCE covered in `grafana-sql-rce-patch` — an unauthenticated or low-privilege attacker can crash Grafana by triggering excessive memory consumption, which is a denial-of-service risk separate from code execution. The existing `grafana-sql-rce-patch` proposal has no acceptance criteria for the DoS scenario, so a gap remains even after that proposal is executed. The fix is the same patched release set (v12.1.10+), making this a low-effort addition that closes the gap with a dedicated acceptance criterion. Note: if Grafana runs in the `labs` tenant, the memory exhaustion surface is doubly relevant given that `labs` is already near its memory quota.
**Source:** https://nvd.nist.gov/vuln/detail/CVE-2026-27880

### 3. argocd-image-updater-ns-escalation: ArgoCD Image Updater Cross-Namespace Escalation (CVE-2026-6388)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-6388 (CVSS 9.1 Critical) allows an attacker with limited permissions to bypass Kubernetes namespace boundaries and trigger unauthorized image updates across tenant applications — a direct threat to the multi-tenant `admins`/`labs` separation the platform relies on. The proposal `argocd-image-updater-ns-escalation` was written to address this but remains unresolved; surfacing it again here ensures it continues to be prioritized ahead of lower-severity items. Disclosure was April 15, making this now three weeks unpatched.
**Source:** https://www.thehackerwire.com/argocd-image-updater-cross-namespace-privilege-escalation-cve-2026-6388/
