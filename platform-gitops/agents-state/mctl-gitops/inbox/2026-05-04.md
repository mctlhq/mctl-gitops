## Top-3 (for spec-writer)

### 1. argocd-serversidediff-secret-leak: ArgoCD CVSS 9.6 — Plaintext Secret Extraction via ServerSideDiff
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-43824 (CVSS 9.6) allows any read-only ArgoCD user to extract plaintext Kubernetes Secret data through the ServerSideDiff endpoint when `IncludeMutationWebhook=true` is active. ArgoCD is the core reconciliation engine for the entire platform and manages secrets across both `admins` and `labs` tenants. Patching to v3.3.9 is a targeted version-pin change with no CRD migrations and closes the highest-severity unaddressed CVE in today's inbox.
**Source:** https://github.com/argoproj/argo-cd/security/advisories/GHSA-3v3m-wc6v-x4x3

### 2. vault-ldap-null-bind-bypass: Vault Terraform Provider LDAP Auth Bypass via Null Bind Default
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2025-13357 affects the Vault Terraform Provider (v4.2.0–v5.4.0) by defaulting `deny_null_bind` to false for LDAP auth, which allows unauthenticated binds against permissive LDAP servers — undermining the Vault authentication chain that secures all platform secrets. Vault is the ClusterSecretStore backend (`vault-backend`) used by External Secrets Operator throughout the platform; an auth bypass here has broad blast radius. The fix is a provider version pin update plus an explicit `deny_null_bind = true` in Terraform LDAP auth configuration, both low-effort changes under `infrastructure/`.
**Source:** https://www.esecurityplanet.com/threats/news-hashicorp-vault-vulnerability/

### 3. loki-ruler-path-traversal: Loki Ruler API Unauthenticated Remote File Read (CVE-2026-21726)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-21726 enables unauthenticated remote file read via double-URL-encoded path traversal against the Loki Ruler API, exposing any file readable by the Loki process — including mounted secrets and service account tokens. Architecture lists `grafana/loki` as a tracked dependency (if templated); if Loki is deployed, this is a critical exposure. The fix is a version-pin update to a patched Loki release; no schema or CRD changes are involved. Note: if the Loki Ruler component is not enabled, operational impact is reduced but the version should still be pinned.
**Source:** https://grafana.com/security/security-advisories/cve-2026-21726/
