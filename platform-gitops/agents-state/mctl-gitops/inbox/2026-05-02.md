## Top-3 (for spec-writer)

### 1. eso-cross-namespace-secret-access: ESO CVE-2026-22822 Cross-Namespace Secret Access via getSecretKey
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-22822 (CVSS 9.3 Critical) allows any namespace that can create an ExternalSecret to call `getSecretKey` and read secrets from a different namespace's Vault path, bypassing the ClusterSecretStore tenant isolation boundary. ESO is the single secret delivery channel for all tenants on this platform; cross-namespace reads break the entire tenant trust model. Cross-checked against all existing ESO proposals (`eso-cve-patch`, `eso-dns-exfil-patch`) — neither covers this CVE. No memory footprint increase expected for `labs`.
**Source:** CVE-2026-22822, CVSS 9.3, published 2026-01-20; https://github.com/external-secrets/external-secrets/security/advisories

### 2. terraform-v1-15-1-patch: Upgrade Terraform to v1.15.1 (crash and panic fixes)
**Category:** performance
**Impact:** 2
**Effort:** 1
**Rationale:** Terraform v1.15.1 fixes a panic in typed modules with no expanded instances and a crash on invalid `action_trigger` blocks — both of which can silently abort infrastructure apply runs under `infrastructure/`. Unpatched panics produce no Terraform error output, making failures hard to diagnose in CI. The upgrade is a drop-in patch with no API changes and no memory impact on any tenant.
**Source:** https://github.com/hashicorp/terraform/releases/tag/v1.15.1
