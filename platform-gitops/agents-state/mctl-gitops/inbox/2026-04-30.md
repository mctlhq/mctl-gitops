## Source: cve

- **CVE-2026-40886 (High, CVSS 7.7) — Argo Workflows controller DoS via unchecked annotation parsing** — Malformed `workflows.argoproj.io/pod-gc-strategy` annotation triggers an unrecovered panic in `podGCFromPod()`, crashing the entire controller; poisoned pod persists across restarts causing a crash loop; affects v3.6.5–4.0.4, fixed in v4.0.5 and v3.7.14. https://dailycve.com/argo-workflows-unchecked-array-index-cve-2026-40886-critical/

- **CVE-2026-31892 (unrated in search results) — Argo Workflows WorkflowTemplate security bypass via podSpecPatch** — Any user who can submit Workflows can bypass all security settings defined in a WorkflowTemplate by including a `podSpecPatch` field, bypassing even `templateReferencing: Strict` mode; fixed in v4.0.2 and v3.7.11. https://advisories.gitlab.com/pkg/golang/github.com/argoproj/argo-workflows/CVE-2026-31892/

- **CVE-2026-28229 (unrated in search results) — Argo Workflows unauthorized WorkflowTemplate access** — Workflow template endpoints allow any client with a bearer token (including `Authorization: Bearer nothing`) to retrieve WorkflowTemplates and ClusterWorkflowTemplates, potentially leaking embedded Secret manifests. https://advisories.gitlab.com/pkg/golang/github.com/argoproj/argo-workflows/v3/CVE-2026-28229/

- **CVE-2026-6388 (Critical, CVSS 9.1) — ArgoCD Image Updater cross-namespace privilege escalation** — Insufficient namespace validation allows an authenticated user with permissions to create or modify an ImageUpdater resource to trigger unauthorized image updates on applications managed by other tenants; affects Red Hat OpenShift GitOps and upstream argocd-image-updater. https://www.thehackerwire.com/argocd-image-updater-cross-namespace-privilege-escalation-cve-2026-6388/

- **CVE-2026-35205 (Critical, CVSS 9.1) — Helm 4 plugin signature verification bypass (fail-open)** — When a plugin's `.prov` provenance file is absent, Helm proceeds with installation without error, treating unsigned plugins as verified; plugin install hooks run with operator privileges; affects Helm v4.0.0–4.1.3, fixed in v4.1.4. https://www.tenable.com/cve/CVE-2026-35205

- **CVE-2026-3605 (unrated in search results) — HashiCorp Vault KVv2 metadata/secret deletion policy bypass** — An authenticated user with glob-based KVv2 policy access can delete secrets they are not authorized to read, leading to potential denial-of-service via secret destruction. https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-3605/

- **CVE-2026-5052 (unrated in search results) — HashiCorp Vault PKI ACME validation SSRF via attacker-controlled DNS** — Vault's PKI engine ACME validation does not reject local targets for http-01 and tls-alpn-01 challenges, allowing an attacker to trick Vault into sending requests to local network addresses and potentially disclose information; affects v1.14.0–1.21.4. https://www.cve.news/cve-2026-5052/

- **CVE-2026-27876 (Critical, CVSS 9.1) — Grafana RCE via SQL Expressions and plugin chaining** — An attacker with Viewer-level permissions and the `sqlExpressions` feature toggle enabled can overwrite a Sqlyze driver or AWS data source config to achieve remote code execution with SSH access to the Grafana host; affects Grafana v11.6.0–12.4.1, fixed in v12.1.10, v12.2.8, v12.3.6, v12.4.2, and v13.0.0+. https://grafana.com/security/security-advisories/cve-2026-27876/

- **CVE-2026-21726 (unrated in search results) — Grafana Loki path traversal bypass via double URL encoding** — Bypasses the prior CVE-2021-36156 fix by using double URL encoding to read arbitrary files through the Ruler API endpoint `/loki/api/v1/rules/{namespace}`. https://grafana.com/security/security-advisories/cve-2026-21726/

## Top-3 (for spec-writer)

### 1. argocd-image-updater-ns-escalation: ArgoCD Image Updater cross-namespace privilege escalation (CVE-2026-6388)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVSS 9.1 Critical — an authenticated user can silently trigger image updates on applications owned by other tenants by exploiting missing namespace validation in ArgoCD Image Updater. In a multi-tenant platform (admins + labs) this breaks tenant isolation at the GitOps layer, the most sensitive control plane we operate. Patching or disabling the component closes the blast radius immediately; the fix is scoped to the Image Updater deployment and does not touch ApplicationSet or the bootstrap chart.
**Source:** https://www.thehackerwire.com/argocd-image-updater-cross-namespace-privilege-escalation-cve-2026-6388/

### 2. vault-kvv2-deletion-bypass: Vault KVv2 secret deletion policy bypass (CVE-2026-3605)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Vault at `secrets.mctl.ai` is the single secret store for all tenants via the `vault-backend` ClusterSecretStore; an authenticated user with any glob-based KVv2 policy can delete secrets they are not permitted to read, enabling a targeted denial-of-service by destroying secrets that ExternalSecrets and workloads depend on. This is a distinct attack surface from CVE-2026-5052 (SSRF, already covered by `vault-ssrf-acme-patch`) and requires its own remediation — either upgrading to the fixed Vault release or tightening all KVv2 glob policies. No new memory-intensive components are introduced, so no labs memory risk.
**Source:** https://advisories.gitlab.com/golang/github.com/hashicorp/vault/CVE-2026-3605/

### 3. grafana-sql-rce-patch: Grafana RCE via SQL Expressions (CVE-2026-27876)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVSS 9.1 Critical RCE exploitable by a Viewer-level account is an extreme severity finding; if Grafana is deployed via our templated Helm charts it runs with cluster-level credentials, making host RCE a stepping stone to broader compromise. The fix is a version upgrade to one of several patched releases (v12.1.10+); as an interim mitigation the `sqlExpressions` feature toggle can be disabled without a full upgrade. Flagged as conditional on Grafana being confirmed in-scope ("if templated" per architecture.md), but the severity warrants pre-emptive tracking.
**Source:** https://grafana.com/security/security-advisories/cve-2026-27876/
