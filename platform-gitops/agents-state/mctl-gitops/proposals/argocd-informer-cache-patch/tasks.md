## Rollback

If the upgrade introduces unexpected behaviour, rollback is a single git operation:

1. **Revert the merge commit** that introduced the version bump:
   ```
   git revert <merge-commit-sha>
   git push origin main
   ```
2. ArgoCD detects the revert as a new desired state and performs a rolling update back to
   the previous image tag within one sync cycle (typically under 5 minutes).
3. Confirm rollback is complete: ArgoCD UI version badge shows the previous version;
   all Applications return to `Synced / Healthy`.
4. Open a post-mortem issue describing what went wrong before re-attempting the upgrade.

No database migrations, CRD schema changes, or persistent-volume mutations are involved,
so there is no data to restore. The rollback is purely a git revert + ArgoCD self-sync.
