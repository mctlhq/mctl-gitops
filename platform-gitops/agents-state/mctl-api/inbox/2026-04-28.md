## Top-3 (for spec-writer)

### 1. pgx-sql-injection-fix: Upgrade pgx to v5.9.2 to fix SQL injection and related CVEs
**Category:** security
**Impact:** 5
**Effort:** 1
**Rationale:** GHSA-j88v-2chj-qfwx is a confirmed SQL injection vulnerability in pgx v5.8 (current version) fixed in v5.9.2; SQL injection in the central platform API is a critical risk. The same upgrade also covers CVE-2026-4427 (panic on negative field length in pgproto3) and CVE-2026-33815/33816 (memory-safety issues), making this a single-patch, multi-CVE resolution.
**Source:** GHSA-j88v-2chj-qfwx (CVE-2025-54236); https://github.com/jackc/pgx/tags

### 2. go-upgrade-stdlib-cves: Upgrade Go runtime from 1.24 to 1.26 to resolve stdlib CVEs
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-32280, CVE-2026-32282, and CVE-2026-32283 cover DoS in certificate chain building, a symlink escape in Root.Chmod, and a crypto/tls DoS via TLS 1.3 key update messages — all addressed only in the Go 1.26.x series; the current 1.24 line is no longer receiving fixes for these. As the platform API handles TLS termination and certificate validation, the crypto/tls DoS in particular is a meaningful availability risk.
**Source:** CVE-2026-32280 / CVE-2026-32282 / CVE-2026-32283; https://go.dev/doc/devel/release

### 3. mcp-go-oauth-upgrade: Upgrade mark3labs/mcp-go from v0.31 to v0.49.0 for OAuth Protected Resource Metadata
**Category:** feature
**Impact:** 3
**Effort:** 2
**Rationale:** The service runs 24 MCP tools over Streamable HTTP and is already committed to mark3labs/mcp-go (ADR 0001); v0.49.0 adds RFC9728 OAuth Protected Resource Metadata discovery and improves OAuth handling for SSE and Streamable HTTP transport, which directly strengthens the authentication surface of the MCP server without any architectural change. The 18-minor-version gap also bundles accumulated transport stability fixes.
**Source:** https://github.com/mark3labs/mcp-go/releases/latest (current: v0.31, new: v0.49.0)
