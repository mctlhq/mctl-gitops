## Top-3 (for spec-writer)

### 1. pgx-security-upgrade: Upgrade pgx from v5.8 to v5.9.2 (SQL injection patch)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-41889 is a confirmed SQL injection vulnerability in pgx v5 prior to v5.9.2. Although the simple protocol is non-default, mctl-api uses pgx for identity and audit-log writes where attacker-controlled input could plausibly reach query parameters. A second memory-safety advisory (CVE-2026-33816) is also open without a fix yet — upgrading to v5.9.2 is the highest-impact, lowest-effort action we can take today. No architectural change required; v5.9.x is backward-compatible with v5.8.
**Source:** CVE-2026-41889; jackc/pgx v5.9.2 release (April 18–19, 2026).

### 2. go-runtime-upgrade: Upgrade Go toolchain from 1.24 to 1.26.3
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** Go 1.26.3 patches 14+ CVEs across two patch releases, including CVE-2026-27140 (CVSS 8.8, arbitrary code execution via cgo at build time), CVE-2026-32283 (TLS DoS), and CVE-2026-33814 (HTTP/2 DoS via CONTINUATION loop). mctl-api serves as the platform entry point handling TLS connections, HTTP/2 traffic, and net/http reverse-proxy logic — all directly affected packages. Additionally, the Go 1.26 "Green Tea" GC provides a 10–40% reduction in GC pause overhead, relevant to mctl-api's real-time MCP streaming. Two-major-version jump requires integration test validation but no API changes.
**Source:** CVE-2026-27140, CVE-2026-32283, CVE-2026-33814; go1.26.3 release (May 7, 2026).

### 3. mcp-go-upgrade: Upgrade mcp-go from v0.31 to v0.54.0
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** At 23 minor versions behind, mcp-go v0.31 misses panic fixes in message handlers and goroutine cleanup, as well as CVE-2026-27896 (case-insensitive field name bypass in MCP response validation). mctl-api's core value is its 24-tool MCP server — a panic or validation bypass here directly impacts platform reliability and security. v0.54.0 also adds OpenTelemetry tracing hooks, enabling production observability for MCP calls. Bump requires re-validating all 24 tools against MCP Inspector (per ADR-0001), which drives the effort score to 3.
**Source:** CVE-2026-27896; mark3labs/mcp-go v0.54.0 release (May 13, 2026); ADR-0001.
