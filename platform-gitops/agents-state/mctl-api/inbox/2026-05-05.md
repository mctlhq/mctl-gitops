## Top-3 (for spec-writer)

### 1. mcp-csrf-origin-validation: CSRF via Missing Origin Header Validation on MCP Streamable HTTP Transport
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-33252 directly targets the Streamable HTTP transport used by mctl-api's MCP server at https://api.mctl.ai/mcp. Without `Origin` header validation, any malicious website can issue cross-site POST requests that invoke our 13 write tools (including `trigger_workflow` and identity tools) against authenticated users' sessions. This is an independent, lower-effort fix that can be applied immediately as middleware ahead of — or in parallel with — the broader v0.51 library upgrade. No existing proposal covers this specific CSRF attack vector.
**Source:** CVE-2026-33252 (CVSS 6.5–7.1), https://nvd.nist.gov/vuln/detail/CVE-2026-33252; inbox finding: `mark3labs/mcp-go (MCP Go SDK) — CVE-2026-33252`.

### 2. mcp-go-v051-upgrade: Upgrade mcp-go from v0.31 to v0.51.0
**Category:** security
**Impact:** 3
**Effort:** 4
**Rationale:** The gap between our current v0.31 and the latest v0.51.0 (released 2026-05-04) spans 20 minor versions and includes potential remediation for CVE-2026-33252, RFC 9728 OAuth Protected Resource Metadata support, CORS controls, tool output schema validation, and `SchemaCache` for stateless deployments. ADR 0001 confirms mark3labs/mcp-go as the accepted library, so upgrading is architecturally aligned. The existing `mcp-go-upgrade-v0.50` proposal targets v0.50; v0.51 is a distinct target with additional security-relevant changes that warrant a fresh proposal. Effort is rated 4 because all 24 tools must be re-validated through MCP Inspector per ADR 0001 consequences.
**Source:** mark3labs/mcp-go v0.51.0 release (2026-05-04), https://github.com/mark3labs/mcp-go/releases/latest; CVE-2026-33252 potential patch; ADR 0001.

### 3. go-toolchain-ace-cve-27140: Arbitrary Code Execution in cmd/go via Malicious SWIG File Names (CVE-2026-27140)
**Category:** security
**Impact:** 3
**Effort:** 3
**Rationale:** CVE-2026-27140 allows arbitrary code execution through the `cmd/go` build tool when processing malicious SWIG file names, a build-time attack surface distinct from the runtime stdlib CVEs already covered by `go-upgrade-stdlib-cves-v2`. Our CI/CD pipeline on ArgoCD/Kubernetes builds mctl-api from source, meaning a compromised dependency or supply-chain artifact with a crafted filename could execute code during the build stage with access to build secrets. Upgrading to Go 1.26.x is required to patch this; the effort is moderate as it involves toolchain version gating and full integration test validation.
**Source:** CVE-2026-27140, RHSA-2026:10217, https://access.redhat.com/errata/RHSA-2026:10217; inbox finding: `golang/go — Multiple CVEs in Go standard library (May 2026)`.
