## Top-3 (for spec-writer)

### 1. pgx-sqli-patch: Patch pgx to v5.9.2 to fix SQL injection CVE
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2025-54236 / GHSA-j88v-2chj-qfwx is a confirmed SQL injection vulnerability in the pgx library we import directly at v5.8. mctl-api's Postgres database holds tenant identities and audit logs, and cross-tenant data leak is the service's known critical failure mode — an exploitable SQL injection here is the highest-severity possible finding. The fix is a targeted minor-version bump to v5.9.2 with no breaking API changes; effort is low but the proposal must confirm and document whether mctl-api uses the simple or extended (prepared statement) protocol, since exploitability differs.
**Source:** https://pkg.go.dev/vuln/list — CVE-2025-54236 / GHSA-j88v-2chj-qfwx; pgx CHANGELOG https://github.com/jackc/pgx/blob/master/CHANGELOG.md

### 2. go-runtime-upgrade: Upgrade Go toolchain from 1.24 to 1.26.x
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** Five stdlib CVEs published April 7, 2026 (CVE-2026-32280, 32281, 32283, 27143, 32289) collectively cover crypto/tls deadlock (can halt the entire API server under load), crypto/x509 chain-building DoS (affects our OIDC JWT verification path via go-oidc), and a compiler memory-corruption issue. All are fixed in Go 1.26.x. The upgrade also closes the gap with Go 1.24 which is behind the current active-support window, reducing future CVE accumulation risk.
**Source:** CVE-2026-32283 https://pkg.go.dev/vuln/GO-2026-4870; CVE-2026-32280 https://pkg.go.dev/vuln/GO-2026-4947; CVE-2026-32281 https://pkg.go.dev/vuln/GO-2026-4946; CVE-2026-27143 https://pkg.go.dev/vuln/GO-2026-4868; CVE-2026-32289 https://pkg.go.dev/vuln/GO-2026-4865; Go releases https://go.dev/dl/

### 3. chi-open-redirect-patch: Patch chi to v5.2.5 to fix open redirect CVE
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** GO-2026-4316 / GHSA-vrw8-fxc6-2r93 is an open redirect in chi's RedirectSlashes middleware via Host Header Injection, classified Moderate severity and published January 14, 2026. We are on chi v5.2.1 and the fix is in v5.2.5 — a pure patch release with no API changes, making effort minimal. An open redirect on a platform API can be abused for phishing and OAuth token hijacking attacks against our PKCE flow.
**Source:** https://github.com/go-chi/chi/security — GO-2026-4316 / GHSA-vrw8-fxc6-2r93; chi releases https://github.com/go-chi/chi/releases/latest

Upgrading 19 minor versions is high effort and ADR 0001 explicitly notes that every bump requires re-validating all 24 MCP tools through MCP Inspector — the feature additions (ListPrompts, ListResources, input schema validation) do not justify the risk without a stronger driver.
- **client-go v0.36 upgrade**: Four-version drift is a maintenance concern but no security CVE is attached to our current version (v0.32); lower priority than the three security fixes above.
- **ArgoCD GHSA-3v3m-wc6v-x4x3 and Argo Workflows CVE-2026-40886/31892/28229**: These affect platform-level ArgoCD and Argo Workflows deployments, not Go packages imported by mctl-api itself. Out of scope for this service's proposals.
- **prometheus/client_golang v1.23.2**: Pure dependency cleanup release; no functional or security change relevant to mctl-api.
- **go-oidc v3.18.0**: No CVEs found against the library itself; the bump is dependency maintenance only and brings no urgent driver.
