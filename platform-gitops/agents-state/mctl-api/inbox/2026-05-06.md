## Top-3 (for spec-writer)

### 1. go-upgrade-1262: Upgrade Go toolchain to 1.26.2 (10 security CVEs)
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** Go 1.26.2 (released 2026-04-07) patches 10 CVEs across crypto/x509 (DoS in chain building and policy validation), crypto/tls (TLS 1.3 deadlock / DoS), html/template (XSS via JS template literal context bypass), os/archive/tar, and — critically — cmd/compile (CVE-2026-27143, CVE-2026-27144: bound-check elimination bypass and interface-conversion overlap skip, risking memory corruption in compiled binaries). We are on Go 1.24, two major versions behind. Existing proposals (go-upgrade, go-upgrade-stdlib-cves-v2, go-toolchain-ace-cve-27140) targeted earlier release trains; none cover this specific release with its compiler-level memory-safety issues. Crypto/tls DoS directly threatens our OIDC JWT validation and Vault mTLS paths.
**Source:** https://github.com/golang/go/tags; https://groups.google.com/g/golang-announce/c/0uYbvbPZRWU/m/7CDdDXqeBwAJ; CVE-2026-32280, CVE-2026-32281, CVE-2026-32283, CVE-2026-32289, CVE-2026-27140, CVE-2026-27143, CVE-2026-27144

### 2. argocd-ghsa-3v3m-patch: Apply argo-cd v3.3.9 patch for critical GHSA-3v3m-wc6v-x4x3
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** argo-cd v3.3.9 (released 2026-04-30) patches GHSA-3v3m-wc6v-x4x3, rated critical. mctl-api fetches the ArgoCD token from Vault and calls ArgoCD APIs to surface application status; a critical vulnerability in ArgoCD can be leveraged against our integration surface (token theft, privilege escalation to the gitops plane). The existing `argocd-token-scope-audit` proposal addresses CVE-2025-55190 (token scoping) — a distinct advisory. GHSA-3v3m-wc6v-x4x3 requires its own assessment and compatibility check for v3.3.9.
**Source:** https://github.com/argoproj/argo-cd/releases/latest; GHSA-3v3m-wc6v-x4x3

### 3. mcp-go-v052-upgrade: Upgrade mark3labs/mcp-go to v0.52.0 (TCP/fd leak fix)
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** mcp-go v0.52.0 fixes a TCP connection / file-descriptor leak that occurs when response bodies are not closed on 404 responses during retry loops in StreamableHTTPServer. We expose 24 MCP tools over Streamable HTTP at /mcp; unclosed connections accumulate under any client that retries, risking fd exhaustion and service unavailability. The existing `mcp-go-v051-upgrade` proposal targeted v0.51 and does not include this fix. No existing proposal covers v0.52.
**Source:** https://github.com/mark3labs/mcp-go/releases/latest
