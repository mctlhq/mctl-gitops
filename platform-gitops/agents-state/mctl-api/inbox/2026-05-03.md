## Top-3 (for spec-writer)

### 1. pgx-sql-injection-fix: Upgrade pgx to v5.9.2 to patch SQL injection CVE
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2025-54236 (GHSA-j88v-2chj-qfwx, Critical) is a SQL injection via placeholder confusion with dollar-quoted string literals when using pgx's simple protocol. mctl-api is currently on v5.8 and is directly affected. Postgres holds tenant identities and audit logs — a SQL injection here could result in cross-tenant data exposure, which is the single most critical failure mode listed in `context/architecture.md` ("Known limitations"). Upgrade to v5.9.2 is a patch-level bump with no API changes.
**Source:** https://github.com/jackc/pgx/security/advisories/GHSA-j88v-2chj-qfwx — CVE-2025-54236, Critical; also CVE-2026-4427 (pgproto3 DoS) fixed in same release.

### 2. go-runtime-cve-upgrade: Upgrade Go runtime from 1.24 (EOL) to 1.26.2
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** Go 1.24 is no longer receiving active security support. Four CVEs are currently unpatched on our runtime: DoS via crypto/tls (CVE-2026-32280, CVE-2026-32283), symlink traversal in internal/syscall/unix (CVE-2026-32282), and a remote code execution in cmd/go via malicious SWIG filenames (CVE-2026-27140). The TLS vulnerabilities are directly relevant because mctl-api serves HTTPS and makes outbound TLS calls to Vault, ArgoCD, and Backstage. Upgrading to 1.26.2 closes all four CVEs and aligns with the latest LTS.
**Source:** https://pkg.go.dev/vuln/list — CVE-2026-32280, CVE-2026-32283, CVE-2026-32282, CVE-2026-27140; all fixed in Go 1.26.2 / 1.25.9.

### 3. mcp-go-upgrade: Upgrade mark3labs/mcp-go from v0.31 to v0.50.0
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** We are 19 minor versions behind on the library that underpins mctl-api's entire MCP server (24 tools, OAuth 2.0 PKCE, Streamable HTTP). The v0.50.0 release introduces opt-in per-tool input schema validation (SEP-1303), which hardens against malformed tool calls. The broader MCP Go ecosystem has active RCE research (OX Security, April 2026), and staying far behind increases exposure to JSON-RPC parsing edge-cases documented in CVE-2026-27896. ADR 0001 already confirms we remain on mark3labs/mcp-go — this is an in-place upgrade, not a library switch.
**Source:** https://github.com/mark3labs/mcp-go/releases/latest — v0.50.0 released 2026-04-30; CVE-2026-27896 ecosystem advisory.
