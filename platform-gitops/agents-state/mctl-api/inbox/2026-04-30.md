## Top-3 (for spec-writer)

### 1. pgx-sqli-cve-2025-54236: Patch pgx to v5.9.2 to fix SQL injection (CVE-2025-54236)
**Category:** security
**Impact:** 5
**Effort:** 1
**Rationale:** pgx is used directly by mctl-api for identity storage and audit logs, making this SQL injection vulnerability (placeholder confusion in simple protocol) a direct, confirmed risk to the service's most sensitive data. The fix is a single dependency bump to v5.9.2 with no API changes required. Severity is classified Low by CVSS but the affected surfaces — identities and audit logs — warrant treating this as the highest priority.
**Source:** CVE-2025-54236 / GHSA-j88v-2chj-qfwx — https://github.com/jackc/pgx/security/advisories/GHSA-j88v-2chj-qfwx

### 2. chi-open-redirect-ghsa-mqqf: Patch chi to v5.2.5 to fix open redirect (GHSA-mqqf-5wvp-8fh8)
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** chi is the HTTP router for all mctl-api endpoints; the open redirect in `RedirectSlashes` middleware (CVSS 4.7) allows an attacker to craft a URL that redirects authenticated users to an attacker-controlled domain, enabling phishing and credential harvesting against platform operators. Bumping from v5.2.1 to v5.2.5 closes the vulnerability and picks up the `RouteHeaders` double-invocation fix as a bonus. No API surface change is expected.
**Source:** GHSA-mqqf-5wvp-8fh8 / GO-2026-4316 — https://github.com/advisories/GHSA-mqqf-5wvp-8fh8

### 3. mcp-go-upgrade-v0.50: Upgrade mcp-go to v0.50.0 and enable input schema validation
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** mcp-go is 19 minor versions behind; v0.50.0 introduces opt-in `WithInputSchemaValidation()` which enforces JSON Schema on all 24 tool arguments before execution, closing a class of malformed-input attacks against MCP write tools. The upgrade also provides the opportunity to audit applicability of CVE-2026-27896 (JSON-RPC field smuggling, confirmed in modelcontextprotocol/go-sdk but unconfirmed for mcp-go) — if the pattern is present, the upgrade or a follow-on patch mitigates it. ADR-0001 explicitly permits upgrading mcp-go; the effort reflects the need to re-validate all 24 tools through MCP Inspector post-upgrade.
**Source:** mark3labs/mcp-go v0.50.0 release — https://github.com/mark3labs/mcp-go/releases/latest; CVE-2026-27896 smuggling advisory — https://advisories.gitlab.com/pkg/golang/github.com/modelcontextprotocol/go-sdk/CVE-2026-27896/
