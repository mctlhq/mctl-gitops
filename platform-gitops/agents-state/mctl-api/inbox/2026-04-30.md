## Top-3 (for spec-writer)

### 1. go-upgrade-stdlib-cves: Go 1.24 EOL + 7 additional stdlib CVEs including RCE (upgrade to Go 1.26.2)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Seven CVEs not covered by the existing `go-upgrade-stdlib-cves` proposal have been published against Go 1.24, including CVE-2026-27140 (RCE via SWIG in cmd/go) and CVE-2026-27143/27144 (unsafe memory in the compiler), both of which are critical in a build pipeline context. Go 1.24 reached EOL on 2026-02-11 and receives no further patches, making the upgrade to go1.26.2 the only remediation. This significantly extends the risk surface of the existing proposal and warrants a v2 revision.
**Source:** https://go.dev/doc/devel/release; https://groups.google.com/g/golang-announce/c/0uYbvbPZRWU (CVE-2026-32289, CVE-2026-32288, CVE-2026-27140, CVE-2026-27143, CVE-2026-27144, CVE-2026-33810)

### 2. mcp-json-smuggling-investigation: Investigate CVE-2026-27896 JSON-RPC field-name smuggling in mark3labs/mcp-go
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-27896 exploits Go's encoding/json case-insensitive field matching to smuggle MCP JSON-RPC fields past WAFs; it is confirmed in the official modelcontextprotocol/go-sdk but its applicability to mark3labs/mcp-go (which mctl-api uses for all 24 MCP tools) is explicitly unresolved. Given that mctl-api exposes 13 write tools with cross-tenant blast radius, the risk of an unverified smuggling path is too high to defer. A bounded investigation (code audit + MCP Inspector replay of crafted payloads) is concrete and actionable.
**Source:** https://advisories.gitlab.com/pkg/golang/github.com/modelcontextprotocol/go-sdk/CVE-2026-27896/

### 3. client-go-version-drift: Upgrade kubernetes/client-go from v0.32 to v0.36
**Category:** refactor
**Impact:** 2
**Effort:** 2
**Rationale:** mctl-api is 4 minor versions behind kubernetes/client-go (v0.32 vs v0.36.0 released 2026-04-22), accumulating compatibility and API-surface drift against the Kubernetes clusters it manages. No direct CVEs exist today, but continued drift increases the cost of each future bump and risks subtle API incompatibilities surfacing under production load. The upgrade path is straightforward (no security urgency, no memory footprint change for the `labs` tenant).
**Source:** https://github.com/kubernetes/client-go/tags (v0.36.0, 2026-04-22)
