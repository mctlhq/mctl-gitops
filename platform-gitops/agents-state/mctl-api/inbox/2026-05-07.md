## Top-3 (for spec-writer)

### 1. pgx-cve-sql-injection: Upgrade pgx to v5.9.2 to fix CVE-2025-54236
**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** pgx v5.8 is a direct dependency carrying a confirmed SQL injection advisory (CVE-2025-54236). Although the vulnerable path requires use of QueryExecModeSimpleProtocol, mctl-api owns the Postgres layer for tenant data and audit logs, so the risk surface is real and the fix is a single patch-level bump with no API changes. Resolving known CVEs in direct dependencies is mandatory hygiene before any other improvement.
**Source:** https://github.com/jackc/pgx/security/advisories/GHSA-j88v-2chj-qfwx

### 2. go-runtime-cve-dos: Upgrade Go toolchain to 1.26.0 to fix CVE-2026-32280/32281/32283
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Three DoS CVEs published 2026-04-07 affect crypto/x509 (unbounded chain building, excessive policy mapping work) and crypto/tls (TLS 1.3 key update flood), all patched in Go 1.26. mctl-api is the HTTPS entry point for all UI/CLI/agent traffic and performs OIDC JWT verification via go-oidc — both code paths traverse the vulnerable crypto stack. An attacker with network access can stall the process with a crafted certificate chain or TLS handshake, making this a high-priority runtime upgrade.
**Source:** https://www.cvedetails.com/vulnerability-list/vendor_id-14185/Golang.html

### 3. mcp-go-upgrade: Upgrade mark3labs/mcp-go to v0.52.0 to fix HTTP body leak
**Category:** performance
**Impact:** 3
**Effort:** 3
**Rationale:** mctl-api is 21 minor versions behind on its MCP library (v0.31 vs v0.52.0), which is the transport layer for all 24 MCP tools exposed to Claude.ai and agent clients. Version v0.52.0 includes an HTTP body leak fix that can cause memory growth under sustained MCP traffic — relevant given the labs tenant memory constraints and the fact that the MCP endpoint handles streaming responses. The new transport-agnostic Handle entry point also improves testability. Each bump requires re-validating 24 tools per ADR 0001, hence effort is rated 3.
**Source:** https://github.com/mark3labs/mcp-go/releases/latest
