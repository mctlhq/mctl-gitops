## Top-3 (for spec-writer)

### 1. pgx-cve-sql-injection: Upgrade pgx to v5.9.2 (fix CVE-2025-54236)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2025-54236 is a confirmed SQL injection via placeholder confusion in the simple query protocol; we use pgx for storing identities and audit logs, which makes the vulnerability critical. The fix is a precise minor version bump (5.8.0 → 5.9.2) without breaking changes in the pgx API. There is no risk of increased memory consumption — the change is only in the query parsing logic.
**Source:** CVE-2025-54236 / GHSA-j88v-2chj-qfwx; https://dailycve.com/pgx-go-sql-injection-via-placeholder-confusion-cve-2025-54236-low-severity/

### 2. go-upgrade: Upgrade Go from 1.24 to a current supported branch (1.26)
**Category:** security
**Impact:** 4
**Effort:** 3
**Rationale:** The Go 1.24 branch has left active support — the last patch 1.24.13 shipped in February 2026, after which security fixes in `crypto/tls`, `crypto/x509`, `archive/tar` were issued only in branches 1.25 and 1.26. Running an unsupported Go branch with unpatched TLS/PKI vulnerabilities is critical for a service with three types of bearer authentication and external TLS connections. The move to Go 1.26 requires verifying dependency compatibility but does not affect the architecture.
**Source:** https://go.dev/doc/devel/release; Go 1.26.2 / 1.25.9 security fixes (2026-04-07)

### 3. chi-security-patch: Upgrade chi/v5 to v5.2.5 (RedirectSlashes security fix)
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** chi v5.2.5 contains a security fix in the `RedirectSlashes` middleware, which we potentially use in the HTTP router as part of the standard chi stack. Lagging 4 patch versions while there is an explicit security fix is unacceptable for a public API endpoint (`api.mctl.ai`). Effort is minimal — patch bump without API breaking changes and without resource consumption growth.
**Source:** https://github.com/go-chi/chi/releases/latest; chi/v5 v5.2.5 (2025-02-05)
