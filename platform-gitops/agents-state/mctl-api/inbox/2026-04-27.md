## Top-3 (для spec-writer)

### 1. pgx-cve-sql-injection: Обновление pgx до v5.9.2 (fix CVE-2025-54236)
**Категория:** security
**Impact:** 5
**Effort:** 2
**Обоснование:** CVE-2025-54236 — подтверждённая SQL-инъекция через placeholder confusion в simple query protocol; мы используем pgx для хранения идентичностей и аудит-логов, что делает уязвимость критичной. Фикс — точечный bump minor-версии (5.8.0 → 5.9.2) без breaking changes в pgx API. Риска увеличения потребления памяти нет — изменение только в логике парсинга запросов.
**Source:** CVE-2025-54236 / GHSA-j88v-2chj-qfwx; https://dailycve.com/pgx-go-sql-injection-via-placeholder-confusion-cve-2025-54236-low-severity/

### 2. go-upgrade: Обновление Go с 1.24 на актуальную поддерживаемую ветку (1.26)
**Категория:** security
**Impact:** 4
**Effort:** 3
**Обоснование:** Ветка Go 1.24 вышла из активной поддержки — последний патч 1.24.13 вышел в феврале 2026, после этого security fixes в `crypto/tls`, `crypto/x509`, `archive/tar` были выпущены только в ветках 1.25 и 1.26. Использование неподдерживаемой ветки Go с неисправленными уязвимостями в TLS/PKI критично для сервиса с тремя типами bearer-аутентификации и внешними TLS-соединениями. Переход на Go 1.26 требует проверки совместимости зависимостей, но не затрагивает архитектуру.
**Source:** https://go.dev/doc/devel/release; Go 1.26.2 / 1.25.9 security fixes (2026-04-07)

### 3. chi-security-patch: Обновление chi/v5 до v5.2.5 (security fix RedirectSlashes)
**Категория:** security
**Impact:** 3
**Effort:** 1
**Обоснование:** chi v5.2.5 содержит security fix в middleware `RedirectSlashes`, который мы потенциально используем в HTTP роутере как часть стандартного chi-стека. Отставание на 4 patch-версии при наличии явного security fix недопустимо для публичного API endpoint (`api.mctl.ai`). Effort минимальный — patch-bump без API breaking changes, без роста потребления ресурсов.
**Source:** https://github.com/go-chi/chi/releases/latest; chi/v5 v5.2.5 (2025-02-05)
