## Source: cve

- **CVE-2026-24046 (High) — Backstage scaffolder symlink path traversal** — Multiple Scaffolder actions (`debug:log`, `fs:delete`) and archive extraction (tar/zip) were vulnerable to symlink-based path traversal allowing read/write/delete of arbitrary files outside the workspace. Fixed in `@backstage/backend-defaults` 0.12.2/0.13.2/0.14.1/0.15.0 and `@backstage/plugin-scaffolder-backend` 2.2.2/3.0.2/3.1.1. https://www.cvedetails.com/cve/CVE-2026-24046/

- **CVE-2026-24048 — Backstage SSRF via redirect in FetchUrlReader** — `FetchUrlReader` followed HTTP redirects when reading from URLs in `backend.reading.allow`, enabling SSRF bypass of the URL allowlist; affects catalog and other backend plugins. https://nvd.nist.gov/vuln/detail/CVE-2026-24048

- **CVE-2026-32235 (Medium, CVSS 5.9) — Backstage auth-backend OAuth redirect URI allowlist bypass** — Experimental OIDC provider in `@backstage/plugin-auth-backend` is vulnerable to redirect URI allowlist bypass when Dynamic Client Registration or CIMD is enabled; patched in plugin-auth-backend 0.27.1. https://dailycve.com/backstage-oauth-redirect-bypass-cve-2026-32235-medium/

- **CVE-2026-32236 (Low) — Backstage auth-backend SSRF in experimental CIMD metadata fetch** — When `auth.experimentalClientIdMetadataDocuments.enabled: true`, CIMD metadata fetch validates the initial hostname against private IP ranges but not after HTTP redirects; patched in `@backstage/plugin-auth-backend` 0.27.1. https://radar.offseq.com/threat/cve-2026-32236-cwe-918-server-side-request-forgery-ca31ec08

- **CVE-2026-29185 (Low, CVSS 2.7) — Backstage integration SCM URL path traversal via encoded sequences** — Path traversal sequences in encoded form in SCM URLs could redirect requests to unintended SCM API endpoints using server-side integration credentials; fixed in `@backstage/integration` 1.20.1. https://nvd.nist.gov/vuln/detail/CVE-2026-29185

- **CVE-2026-23869 (High, CVSS 7.5) — React Server Components DoS via crafted HTTP requests** — Specially crafted HTTP requests to Server Function endpoints cause excessive CPU usage; does not affect deployments that do not use React Server Components. https://www.sentinelone.com/vulnerability-database/cve-2026-23869/

- **CVE-2026-23864 — React/Next.js Denial of Service via memory exhaustion** — Malicious HTTP request to a Server Functions endpoint triggers an infinite loop causing CPU hang and OOM; fixed in React 19.0.4, 19.1.5, 19.2.4. https://www.akamai.com/blog/security-research/cve-2026-23864-react-nextjs-denial-of-service

- **CVE-2026-41238 (Medium, CVSS 6.9) — DOMPurify prototype pollution XSS bypass** — Affects DOMPurify 3.0.1–3.3.3; prior prototype pollution gadget can inject permissive regex values into `Object.prototype` causing DOMPurify to allow arbitrary event-handler attributes through sanitization; fixed in 3.4.0. https://www.thehackerwire.com/vulnerability/CVE-2026-41238/

- **CVE-2026-21710 (High) — Node.js DoS via `__proto__` header name in `req.headersDistinct`** — Denial of service via prototype pollution in HTTP header parsing; addressed in Node.js March 2026 security release (v22.22.2). https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

- **CVE-2026-21711 (Medium) — Node.js permission model UDS bind/listen bypass** — Unix Domain Socket server bind/listen works without `--allow-net` when permission model is enabled. https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

## Top-3 (for spec-writer)

### 1. scaffolder-symlink-traversal: Patch Backstage scaffolder symlink path traversal (CVE-2026-24046)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** mctl-portal actively uses the Backstage scaffolder for service onboarding, making this a direct and high-severity attack surface. A malicious template author or compromised template could read, write, or delete arbitrary files outside the workspace via symlink injection in `debug:log`, `fs:delete`, or archive extraction actions. The fix is a targeted package upgrade to `@backstage/backend-defaults` and `@backstage/plugin-scaffolder-backend` with no breaking API changes expected.
**Source:** CVE-2026-24046 — https://www.cvedetails.com/cve/CVE-2026-24046/

### 2. nodejs-security-upgrade: Upgrade Node.js runtime to v22.22.2 (CVE-2026-21710 + 7 more)
**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** The current runtime is Node.js 22 (engines.node: "22 || 24") and the v22.22.2 LTS security release addresses eight CVEs, including CVE-2026-21710 (High severity DoS via prototype pollution in HTTP header parsing) which directly threatens the Backstage backend's HTTP server availability. This is an infrastructure-level fix with minimal upgrade risk since we stay within the same major version.
**Source:** Node.js v22.22.2 LTS — https://github.com/nodejs/node/releases; CVE-2026-21710 — https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

### 3. fetchurl-ssrf: Patch Backstage SSRF via FetchUrlReader redirect (CVE-2026-24048)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** mctl-portal uses the catalog plugin extensively and the proxy plugin for external API integrations — both rely on `FetchUrlReader` for backend URL reads. An attacker who can influence catalog entity URLs or proxy targets could bypass the `backend.reading.allow` allowlist via HTTP redirects to reach internal services (e.g., mctl-api, Vault, Prometheus). Fixing this requires upgrading the affected Backstage backend package and auditing the current `backend.reading.allow` configuration.
**Source:** CVE-2026-24048 — https://nvd.nist.gov/vuln/detail/CVE-2026-24048
