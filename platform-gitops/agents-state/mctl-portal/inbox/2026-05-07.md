## Top-3 (for spec-writer)

### 1. backstage-v1504-security-upgrade: Upgrade Backstage to v1.50.4 (security patch)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Backstage v1.50.4 is a dedicated security release that resolves two High-severity CVEs (CVE-2026-24046 path traversal in Scaffolder, CVE-2026-24048 SSRF bypass in FetchUrlReader) and two additional CVEs (Medium, Low). The Scaffolder is mctl-portal's primary onboarding mechanism — symlink path traversal by any user with template access poses a critical risk to mctl-gitops workspace files. All four CVEs are resolved in a single version bump with no breaking changes noted.
**Source:** https://github.com/backstage/backstage/releases/tag/v1.50.4 | https://nvd.nist.gov/vuln/detail/CVE-2026-24046 | https://nvd.nist.gov/vuln/detail/CVE-2026-24048

### 2. scaffolder-secret-log-redaction: Redact secrets from Scaffolder execution logs
**Category:** security
**Impact:** 3
**Effort:** 3
**Rationale:** GHSA-3x3q-ghcp-whf7 discloses that the `fetch:template` Scaffolder action can emit Vault-backed secret values in execution logs, which are persisted in Loki and accessible to all operators. mctl-portal's Scaffolder uses Vault-backed secrets for onboarding new services; log leakage would expose those secrets to anyone with log access. This mitigation is independent of the version upgrade and provides defence-in-depth.
**Source:** https://github.com/backstage/backstage/security/advisories/GHSA-3x3q-ghcp-whf7

### 3. nodejs-lts-v24-baseline: Adopt Node.js v24 LTS as the baseline runtime
**Category:** performance
**Impact:** 2
**Effort:** 2
**Rationale:** The portal already declares `"node": "22 || 24"` in engines, but the Docker base image and CI target v22. Node.js v24 ('Krypton') is the current Active LTS with security-relevant additions (raw key format in crypto, new HTTP/2 config options) and performance improvements. Standardising on v24 closes the gap between the declared engines field and the actual runtime, and avoids Node 22 reaching EOL in April 2027.
**Source:** https://github.com/nodejs/node/releases/tag/v24.15.0
