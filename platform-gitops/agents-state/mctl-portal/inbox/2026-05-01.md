- **CVE-2026-21637 (Node.js)** — CVSS HIGH; `SNICallback` invocations lack `try/catch`, allowing synchronous exceptions on unexpected `servername` input to crash the process; affects Node 20.x, 22.x, 24.x, 25.x; disclosed in the March 24, 2026 Node.js security release. https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

- **CVE-2026-21710 (Node.js)** — CVSS HIGH; accessing `req.headersDistinct` when a request contains a header named `__proto__` throws an uncaught `TypeError` that crashes the process (DoS); affects Node 20.x, 22.x, 24.x, 25.x; disclosed March 24, 2026. https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

- **CVE-2026-21713 (Node.js)** — CVSS MEDIUM; non-constant-time `memcmp()` in HMAC verification creates a timing side-channel that could allow MAC forgery; affects Node 20.x, 22.x, 24.x, 25.x; disclosed March 24, 2026. https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

- **CVE-2026-21714 (Node.js)** — CVSS MEDIUM; `WINDOW_UPDATE` frames on stream 0 in HTTP/2 servers prevent `Http2Session` objects from being garbage collected, causing memory exhaustion; affects Node 20.x, 22.x, 24.x, 25.x; disclosed March 24, 2026. https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

- **CVE-2026-21717 (Node.js / V8)** — CVSS MEDIUM; integer-like strings hash to predictable values in V8 enabling HashDoS attacks, most easily triggered via `JSON.parse()` on attacker-controlled input; affects Node 20.x, 22.x, 24.x, 25.x; disclosed March 24, 2026. https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

## Top-3 (for spec-writer)

### 1. backstage-catalog-security-patch: Backstage v1.50.4 Security Patch for Catalog Plugins
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Backstage v1.50.4 is a security-only release (April 29, 2026) patching vulnerabilities in the catalog plugin family (`plugin-catalog-backend-module-unprocessed`, `plugin-catalog-unprocessed-entities-common`, `plugin-catalog-unprocessed-entities`). These catalog plugins are part of our active plugin set and no existing proposal covers this release. The ADR-0001 rule to wait ~1 week after a Backstage release for community-plugins compat has now been satisfied (release April 29, evaluation May 1 — note: only 2 days elapsed; the spec should schedule deployment no earlier than May 6 to honor the 1-week window). No labs memory risk; mctl-portal runs under admins only.
**Source:** https://github.com/backstage/backstage/releases/tag/v1.50.4

### 2. nodejs-process-crash-cves: Node.js March 2026 Process-Crash CVEs (21637 + 21713 + 21714 + 21717)
**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** CVE-2026-21637 (SNICallback crash, HIGH), CVE-2026-21713 (HMAC timing side-channel, MEDIUM), CVE-2026-21714 (HTTP/2 memory exhaustion, MEDIUM), and CVE-2026-21717 (HashDoS via V8, MEDIUM) were all disclosed in the March 24, 2026 Node.js security release and affect Node 22.x. The existing `nodejs-security-upgrade` proposal (slug: nodejs-security-upgrade) addresses CVE-2026-21710 and states "eight CVEs" in its requirements, implying these are already in scope — but the proposal title and design reference only the `__proto__` header crash. A new proposal is warranted to ensure each crash scenario (especially the SNICallback process kill) has explicit acceptance criteria and test coverage, and to confirm the pinned image version resolves all five HIGH/MEDIUM CVEs from this release batch. No labs impact; mctl-portal is admins-only.
**Source:** https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

### 3. typescript-6-upgrade: Upgrade TypeScript to v6.0.3
**Category:** refactor
**Impact:** 2
**Effort:** 3
**Rationale:** TypeScript 6.0.3 is now the stable release on the 6.0 major branch and introduces type-system improvements and stricter checks that reduce entire classes of runtime bugs at compile time. No existing proposal covers this upgrade, and Backstage's Yarn-workspace structure means the upgrade must be validated across all workspace packages. This is a non-urgent DX improvement but provides long-term type-safety dividends; it should be scheduled after the security items above are resolved to avoid compounding risk.
**Source:** https://github.com/microsoft/TypeScript/releases/tag/v6.0.3

- **Yarn v4.14.1:** EBADF fix targets Node 24.15+; mctl-portal runs Node 22. Not applicable to current stack.
