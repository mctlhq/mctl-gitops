## Top-3 (for spec-writer)

### 1. scaffolder-cve-2026-24046-path-traversal: Fix Scaffolder symlink path traversal (CVE-2026-24046)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-24046 is a HIGH-severity (CVSS 7.1) symlink path traversal in Backstage Scaffolder that lets any user with template execution access read, write, or delete files outside the task workspace — a direct threat given that `scaffolder` is a core plugin used for service onboarding. The fix is a targeted patch upgrade of `@backstage/plugin-scaffolder-backend` (to 2.2.2/3.0.2/3.1.1) and `@backstage/backend-defaults` (to 0.15.0), both within the current 1.50.x minor line, carrying minimal compat risk. CVE-2026-24048 (SSRF in FetchUrlReader, CVSS 3.5) is fixed by the same `@backstage/backend-defaults` bump and should be resolved in the same change.
**Source:** https://nvd.nist.gov/vuln/detail/CVE-2026-24046 ; https://nvd.nist.gov/vuln/detail/CVE-2026-24048

### 2. nodejs-22-security-upgrade: Upgrade Node.js runtime to v22.22.2
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Node.js v22.22.2 (2026-03-24) patches 8 CVEs against the v22 LTS line we run, including two high-severity process crashes — SNICallback exception bypass (CVE-2026-21637) and `headersDistinct` prototype pollution DoS (CVE-2026-21710) — either of which can bring down the portal pod. The January 2026 batch also introduced high-severity issues (buffer memory leak, HTTP/2 DoS) that require v22.22.0+, so the service is currently exposed to multiple unpatched runtime vulnerabilities. Updating the base Docker image tag and the CI Node.js version pin is low-effort.
**Source:** https://nodejs.org/en/blog/vulnerability/march-2026-security-releases ; https://nodejs.org/en/blog/vulnerability/december-2025-security-releases

### 3. backstage-1-50-4-patch: Upgrade Backstage to v1.50.4 and patch catalog security modules
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** Backstage v1.50.4 (2026-04-29) is a security-fix patch release on the same 1.50.x minor line, making it safe to apply without community-plugins compat concerns (ADR-0001 only cautions against bumping major on release day). It patches `@backstage/plugin-catalog-backend-module-unprocessed` and related catalog packages that are in active use, and also resolves CVE-2026-29185 (path traversal in `@backstage/integration` 1.20.1, CVSS 2.7) in the same upgrade sweep. Running a stale patch version against published CVEs is an unnecessary compliance risk.
**Source:** https://github.com/backstage/backstage/releases/tag/v1.50.4 ; https://nvd.nist.gov/vuln/detail/CVE-2026-29185
