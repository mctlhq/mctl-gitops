## Top-3 (for spec-writer)

### 1. scaffolder-path-traversal: Close CVE-2026-24046 — symlink path traversal in scaffolder
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** The vulnerability allows an authenticated user, via a crafted symlink in a template, to read, write, and delete arbitrary files outside the workspace on the Backstage server — including secrets mounted from Vault via ExternalSecret. Scaffolder is our central onboarding tool and is heavily used by developers, which makes the attack surface concrete. The fix is precise: upgrade `@backstage/backend-defaults` to 0.12.2+ and `plugin-scaffolder-backend` to 3.1.1+, which fits a routine patch upgrade.
**Source:** CVE-2026-24046 — https://www.cvedetails.com/cve/CVE-2026-24046/

### 2. scaffolder-secret-leak: Close CVE-2026-32237 — env secret leak via dry-run
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** An authenticated user with template dry-run permission obtains server-side env variables through incomplete redaction of nested JSON objects in the endpoint response. Our backend reads secrets from Vault via ExternalSecret and exposes them as env, which makes the leak materially sensitive (Vault tokens, Postgres DSN, GitHub App credentials). Affects versions 3.1.0–3.1.4, fix in the same 3.1.1+, meaning a single upgrade closes both CVEs (#1 and #2 ship in the same PR).
**Source:** CVE-2026-32237 — https://dailycve.com/backstage-plugin-scaffolder-backend-information-exposure-cve-2026-32237-moderate/

### 3. integration-scm-credentials: Close CVE-2026-29185 — SCM credential leak via path traversal in URLs
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Encoded path-traversal sequences in user-supplied SCM URLs (catalog-import, scaffolder git actions, github-actions plugin) redirect requests to arbitrary SCM API endpoints with server-side credentials — in our case, the GitHub App token and the Bitbucket integration may be compromised. Fix: upgrade `@backstage/integration` to v1.20.1. Backstage v1.50.3 (patch dated 22 April) most likely includes a compatible integration version, so upgrading Backstage to the latest patch closes this vector as well.
**Source:** CVE-2026-29185 — https://github.com/backstage/backstage/security/advisories/GHSA-95v5-prp4-5gv5

## Dropped

- **CVE-2026-23864 (React RSC DoS)** — Backstage does not use React Server Components, the vulnerability concerns only Server Functions endpoints (Next.js and similar). Not applicable.
- **CVE-2026-22029 (React Router XSS)** — no data on the affected React Router version in our stack; Backstage uses its own routing layer, no documented direct open redirect pattern. Requires a separate audit, but does not make the top without version confirmation.
- **CVE-2025-59828 (Yarn 2+ auth bypass)** — we use yarn classic v1.22.x, the vulnerability concerns only the Yarn 2.0+ plugin mechanism. Not applicable.
- **Node.js v24.15.0 LTS** — `engines` already contains `"22 || 24"`, the move is possible but not urgent; no security trigger. Candidate for a separate DX proposal.
- **Node.js v22.22.1 patch** — `--heapsnapshot-near-heap-limit` going stable is useful for diagnostics but not critical; the current v22.x already covers the security baseline. Nice-to-have.

- **backstage v1.50.3 (standalone)** — performance and UI fixes are useful but lack a standalone security trigger; will be included as part of the patch upgrade closing CVE-2026-29185 (#3 in the top).
- **TypeScript v6.0** — major; requires a full compat audit of all plugins and custom code. A separate proposal, not a patch-day task.
- **Playwright v1.59.x** — screencast features and the Windows regression do not affect Linux CI/CD; DX improvement, not security.
- **Prettier v3.8.x** — SCSS and Angular fixes are not relevant to our stack (no Angular, SCSS `if()` not used). Nice-to-have.
