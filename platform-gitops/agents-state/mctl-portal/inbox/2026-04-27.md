## Top-3 (для spec-writer)

### 1. scaffolder-path-traversal: Закрыть CVE-2026-24046 — symlink path traversal в scaffolder
**Категория:** security
**Impact:** 5
**Effort:** 2
**Обоснование:** Уязвимость позволяет аутентифицированному пользователю через crafted symlink в шаблоне читать, писать и удалять произвольные файлы за пределами workspace на сервере Backstage — включая секреты, монтированные из Vault через ExternalSecret. Scaffolder у нас является центральным инструментом онбординга и активно используется разработчиками, что делает поверхность атаки реальной. Фикс точечный: обновить `@backstage/backend-defaults` до 0.12.2+ и `plugin-scaffolder-backend` до 3.1.1+, что укладывается в штатный patch upgrade.
**Source:** CVE-2026-24046 — https://www.cvedetails.com/cve/CVE-2026-24046/

### 2. scaffolder-secret-leak: Закрыть CVE-2026-32237 — утечка env-секретов через dry-run
**Категория:** security
**Impact:** 4
**Effort:** 2
**Обоснование:** Аутентифицированный пользователь с правом на dry-run шаблонов получает серверные env-переменные через неполную редакцию вложенных JSON-объектов в ответе эндпоинта. У нас backend читает секреты из Vault через ExternalSecret и пробрасывает их как env, что делает утечку реально чувствительной (Vault tokens, Postgres DSN, GitHub App credentials). Затронуты версии 3.1.0–3.1.4, фикс в том же 3.1.1+, что означает одно обновление закрывает оба CVE (#1 и #2 идут в одном PR).
**Source:** CVE-2026-32237 — https://dailycve.com/backstage-plugin-scaffolder-backend-information-exposure-cve-2026-32237-moderate/

### 3. integration-scm-credentials: Закрыть CVE-2026-29185 — утечка SCM credentials через path traversal в URLs
**Категория:** security
**Impact:** 4
**Effort:** 2
**Обоснование:** Закодированные path-traversal последовательности в user-supplied SCM URLs (catalog-import, scaffolder git actions, github-actions plugin) перенаправляют запросы к произвольным SCM API-эндпоинтам с серверными credentials — в нашем случае GitHub App token и Bitbucket-интеграция могут быть скомпрометированы. Фикс: обновить `@backstage/integration` до v1.20.1. Backstage v1.50.3 (patch от 22 апреля) скорее всего включает совместимую версию integration, поэтому обновление Backstage до последнего патча закрывает и этот вектор.
**Source:** CVE-2026-29185 — https://github.com/backstage/backstage/security/advisories/GHSA-95v5-prp4-5gv5

## Отброшено

- **CVE-2026-23864 (React RSC DoS)** — Backstage не использует React Server Components, уязвимость касается только Server Functions endpoints (Next.js и аналоги). Нерелевантно.
- **CVE-2026-22029 (React Router XSS)** — нет данных о затронутой версии React Router в нашем стеке; Backstage использует свой routing layer, прямой open redirect pattern не документирован. Требует отдельного audit, но не попадает в топ без подтверждения версии.
- **CVE-2025-59828 (Yarn 2+ auth bypass)** — мы используем yarn classic v1.22.x, уязвимость касается только Yarn 2.0+ plugin mechanism. Нерелевантно.
- **Node.js v24.15.0 LTS** — `engines` уже содержит `"22 || 24"`, переход возможен, но не срочен; нет security-триггера. Candidate для отдельного DX proposal.
- **Node.js v22.22.1 патч** — `--heapsnapshot-near-heap-limit` stable полезен для диагностики, но не критичен; текущая v22.x уже покрывает security baseline. Nice-to-have.
- **Backstage v1.51.0-next.0** — pre-release, ADR 0001 явно запрещает апгрейд major/minor на patch-day без ожидания compat community-plugins.
- **backstage v1.50.3 (standalone)** — performance и UI fixes полезны, но без самостоятельного security-триггера; будет включён как часть patch-upgrade при закрытии CVE-2026-29185 (#3 в топе).
- **TypeScript v6.0** — major, требует полного compat audit всех плагинов и custom кода. Отдельный proposal, не patch-day задача.
- **Playwright v1.59.x** — screencast features и Windows регрессия не затрагивают CI/CD на Linux; DX улучшение, не security.
- **Prettier v3.8.x** — SCSS и Angular fixes нерелевантны для нашего стека (нет Angular, SCSS `if()` не используется). Nice-to-have.
- **Yarn v1.22.22** — дата релиза март 2024, уже должна быть в использовании или не давать ценности.
