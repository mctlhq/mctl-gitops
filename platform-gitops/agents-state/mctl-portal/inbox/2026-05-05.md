## Top-3 (for spec-writer)

### 1. techdocs-mkdocs-code-exec: TechDocs MkDocs allowlist bypass enabling arbitrary Python code execution
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-29186 (CVSS 7.7) allows any authenticated user to execute arbitrary Python code on the backend by crafting a malicious mkdocs.yml, bypassing all TechDocs security controls. mctl-portal runs TechDocs and this is distinct from the previously proposed techdocs-path-traversal-fix (which addressed a path traversal vector, not MkDocs config allowlist escape and code execution). Upgrading plugin-techdocs-node to v1.14.3 is the required fix.
**Source:** https://github.com/backstage/backstage/security/advisories/GHSA-6jr7-99pf-8vgf — CVE-2026-29186, CVSS 7.7 High.

### 2. auth-backend-redirect-bypass: OAuth redirect URI allowlist bypass in plugin-auth-backend
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-32235 (CVSS 5.9) allows a crafted OAuth redirect URI to pass allowlist validation and exfiltrate authorization codes to an attacker-controlled host. mctl-portal relies on plugin-auth-backend for all Dex JWT authentication, making a bypass here a critical auth-theft risk for the entire platform. Patching to plugin-auth-backend v0.27.1 eliminates this exposure.
**Source:** https://advisories.gitlab.com/pkg/npm/@backstage/plugin-auth-backend/CVE-2026-32235/ — CVE-2026-32235, CVSS 5.9 Medium.

### 3. auth-backend-metadata-ssrf: SSRF via unvalidated redirect in auth-backend CIMD metadata fetch
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** CVE-2026-32236 is a low-severity SSRF companion to CVE-2026-32235 in the same package; when experimentalClientIdMetadataDocuments is enabled the metadata fetch follows redirects without revalidating against the allowlist. Although the same v0.27.1 patch resolves both CVEs, this is tracked separately because it represents a distinct vulnerability class (SSRF vs. auth-code theft) and the spec-writer needs explicit requirements to verify the experimental flag is either disabled or safe. Bundling without explicit requirements risks the SSRF vector being overlooked in testing.
**Source:** https://advisories.gitlab.com/pkg/npm/@backstage/plugin-auth-backend/CVE-2026-32236/ — CVE-2026-32236, Low severity.
