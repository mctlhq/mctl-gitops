## Top-3 (for spec-writer)

### 1. scaffolder-path-traversal-fix: Patch Backstage Scaffolder symlink path traversal (CVE-2026-24046)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** The Scaffolder is actively used for service onboarding and is directly exposed to users who can author and execute templates. A CVSS 7.1 path traversal allows reading arbitrary backend files, deleting workspace files, and writing outside the workspace via malicious symlinks in tar/zip archives — all without any additional privileges. Patched packages are already available and the fix is a dependency version bump.
**Source:** https://github.com/backstage/backstage/security/advisories/GHSA-rq6q-wr2q-7pgp

### 2. nodejs-22-security-update: Upgrade Node.js runtime to v22.22.3
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** Node.js v22.22.3 resolves multiple runtime-level vulnerabilities including a Zlib use-after-free, an HTTP2 FileHandle leak, and a URL parser crash via malformed UNC hostnames — all of which affect any Node.js 22 service regardless of application code. Since mctl-portal runs on Node.js 22 and is served via nginx+Docker, updating the base image pins the fix with minimal risk. OpenSSL is also upgraded to 3.5.6 and root certificates refreshed.
**Source:** https://nodejs.org/en/blog/release/v22.22.3

### 3. techdocs-path-traversal-fix: Patch Backstage TechDocs local generator path traversal (CVE-2026-25152)
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** mctl-portal uses the TechDocs plugin for markdown documentation alongside services. If the local MkDocs generator is enabled (common in CI or self-hosted setups), symlinks in a docs directory allow arbitrary host files to be embedded into generated HTML output, leaking server-side content to portal users. The fix is a patch upgrade to `@backstage/plugin-techdocs-node` 1.14.1 or 1.13.11.
**Source:** https://github.com/backstage/backstage/security/advisories/GHSA-w669-jj7h-88m9
