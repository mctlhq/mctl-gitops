## Source: cve

- **CVE-2026-29185 — Backstage @backstage/integration SCM URL path traversal** — Encoded path traversal sequences in user-supplied SCM URLs can redirect server-side integration credential requests to unintended API endpoints; affects GitHub, Bitbucket Server, and Bitbucket Cloud integrations. Fix: upgrade `@backstage/integration` to 1.20.1+. https://advisories.gitlab.com/pkg/npm/@backstage/integration/CVE-2026-29185/

- **CVE-2026-32236 — Backstage @backstage/plugin-auth-backend SSRF** — SSRF in the experimental CIMD metadata fetch: redirect validation is applied to the initial `client_id` hostname but not after HTTP redirects; requires `auth.experimentalClientIdMetadataDocuments.enabled: true` (off by default). Fix: upgrade to 0.27.1+. https://advisories.gitlab.com/pkg/npm/@backstage/plugin-auth-backend/CVE-2026-32236/

- **CVE-2026-23947 — Backstage @backstage/plugin-techdocs-node path traversal** — When `techdocs.generator.runIn: local`, symlinks inside the docs directory are followed by MkDocs, allowing file contents from the host filesystem to be embedded in generated HTML and exposed to documentation viewers. Fix: upgrade to 1.13.11 / 1.14.1+. (Source: web search / GitLab advisory database)

- **CVE-2026-23869 — React Server Components Denial of Service (CVSS 7.5)** — Disclosed April 8; crafted HTTP requests to Server Function endpoints cause excessive CPU usage for up to a minute before throwing a catchable error. Affects next, react-router, waku, @parcel/rsc, and other RSC frameworks. https://www.sentinelone.com/vulnerability-database/cve-2026-23869/

- **CVE-2026-23864 — React/Next.js Denial of Service via memory exhaustion** — Malicious HTTP request to any Server Functions endpoint triggers infinite loop, causing server hang, OOM, or excessive CPU; patched in React 19.0.4, 19.1.5, 19.2.4. https://www.akamai.com/blog/security-research/cve-2026-23864-react-nextjs-denial-of-service

## Top-3 (for spec-writer)

### 1. techdocs-path-traversal-fix: Patch TechDocs path traversal (CVE-2026-23947)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-23947 directly matches our exact configuration: `techdocs.generator.runIn: local` is in use, meaning symlinks in any docs directory can expose arbitrary host filesystem files to documentation viewers. This is an unambiguous, high-severity vulnerability with a straightforward fix (upgrade `@backstage/plugin-techdocs-node` to 1.13.11 or 1.14.1+). The fix should be bundled with the Backstage v1.50.3 patch upgrade, which also resolves the facets performance regression.
**Source:** CVE-2026-23947 — https://advisories.gitlab.com/pkg/npm/@backstage/plugin-techdocs-node/CVE-2026-23947/

### 2. integration-credential-leak-fix: Patch Backstage integration SCM credential leak (CVE-2026-29185)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-29185 affects `@backstage/integration`, which underpins the GitHub integration and PR/issue widgets used by mctl-portal. Encoded path traversal in SCM URLs can redirect server-side credential requests (GitHub tokens) to attacker-controlled endpoints. Upgrading `@backstage/integration` to 1.20.1+ closes the vulnerability; this should be done in the same patch cycle as finding #1.
**Source:** CVE-2026-29185 — https://advisories.gitlab.com/pkg/npm/@backstage/integration/CVE-2026-29185/

### 3. nodejs-security-patch: Apply Node.js March 2026 security fixes
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** The March 2026 Node.js security release covers three CVEs against the Node.js 22 runtime we run in production: a `__proto__` header Denial-of-Service (CVE-2026-21710), a permission-model bypass (CVE-2026-21711), and an SNI error-handling regression (CVE-2026-21637 follow-up). Patching the base image or runtime pin to the latest Node.js 22 LTS security release closes all three at once. This is independent of the Backstage package upgrades and should be tracked separately.
**Source:** Node.js March 2026 security releases — https://nodejs.org/en/blog/vulnerability/march-2026-security-releases

## Dropped
- **CVE-2026-32236 (SSRF in plugin-auth-backend):** Requires `auth.experimentalClientIdMetadataDocuments.enabled: true`, which is off by default. No evidence this flag is enabled; risk is effectively zero unless confirmed otherwise.
- **CVE-2026-23869 / CVE-2026-23864 (React RSC DoS):** Affects React Server Components frameworks (Next.js, react-router, waku). Backstage uses React as a client-side SPA and does not use RSC or Server Functions. Not applicable.
- **CVE-2026-2441 (Chromium/Playwright zero-day):** Playwright is used only for e2e tests in CI, not in a production internet-facing context. Risk is real but confined to developer/CI machines. Warrants a routine Playwright upgrade but does not meet the Top-3 threshold against three production runtime CVEs.
- **Backstage v1.50.3 facets performance fix:** Valuable but purely a performance/DX improvement. Will be delivered automatically when applying the security upgrades in findings #1 and #2; no standalone entry needed.
- **TypeScript v6.0.3:** New major with no direct CVE. A DX/refactor proposal is reasonable but lower priority than three open security issues. Revisit once the security backlog is clear.
- **Node.js v25.9.0:** Current-release line, not LTS. We target Node.js 22 LTS; upgrading to v25 is out of scope and architecturally not justified today.
- **Playwright v1.59.1:** Single Windows console revert; no security impact; not relevant to Linux-based CI.
- **Prettier v3.8.3:** SCSS `if()` trailing comma fix; minimal relevance to portal codebase.
- **Yarn v4.14.1:** EBADF fstat fix targets Node 24.15+; we run Node 22, so not applicable.
