## Top-3 (for spec-writer)

### 1. scaffolder-symlink-traversal: Backstage Scaffolder symlink path traversal (CVE-2026-24046)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** High-severity CVE directly affecting the scaffolder plugin, which is a core component used for service onboarding into mctl-gitops/Argo Workflows. An authenticated user who can create or execute templates can read, delete, or overwrite arbitrary files outside the workspace — a critical blast radius in a multi-tenant portal environment. The fix is a targeted patch-version bump of `@backstage/plugin-scaffolder-backend` and `@backstage/backend-defaults` with no breaking changes.
**Source:** https://www.cvedetails.com/cve/CVE-2026-24046/

### 2. fetchurlreader-ssrf: Backstage FetchUrlReader SSRF via HTTP redirect (CVE-2026-24048)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** FetchUrlReader is used by catalog-import, TechDocs, and the proxy plugin — all active in mctl-portal. The SSRF allows any host in the `backend.reading.allow` list to redirect requests to internal URLs that are not on the allowlist, effectively bypassing the primary URL security boundary. Combined with the broader Backstage v1.50.4 security patch release (which also addresses catalog-unprocessed-entities vulnerabilities), this argues for a single coordinated Backstage patch upgrade.
**Source:** https://advisories.gitlab.com/pkg/npm/@backstage/backend-defaults/CVE-2026-24048/; https://github.com/backstage/backstage/releases/tag/v1.50.4

### 3. nodejs22-security-batch: Node.js 22 March 2026 security upgrade (v22.22.2)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** mctl-portal runs on Node.js 22 and is currently behind v22.22.2, which patches two High-severity CVEs: a TLS SNI Denial-of-Service (CVE-2026-21637) and a `__proto__` header DoS (CVE-2026-21710), both capable of crashing the backend process. Bumping the base Docker image tag resolves all seven CVEs in the batch with minimal risk; the undici 6.24.1 update is also included, which benefits the Backstage backend's outbound HTTP client.
**Source:** https://nodejs.org/en/blog/vulnerability/march-2026-security-releases
