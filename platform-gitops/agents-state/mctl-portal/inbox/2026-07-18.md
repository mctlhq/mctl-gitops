## Top-3 (for spec-writer)

### 1. nodejs-june-2026-security-release: Patch Node.js runtime for the June 2026 security batch (CVE-2026-48933, CVE-2026-48618, +2 medium)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** Two High-severity CVEs (WebCrypto `subtle.encrypt()` integer overflow causing remote DoS; TLS hostname-verification bypass via Unicode dot separator) directly hit our `engines.node: "22 || 24"` target and are reachable on a public-facing, auth-fronted service (Dex JWT over TLS). This is a genuinely new batch — checked against all 8 existing `nodejs-*` proposals in `proposals/`, which only cover the March 2026 batch (CVE-2026-2163x/2171x) and the May 2026 batch (v22.22.3, Zlib/HTTP2/URL-parser); none mention v22.23.0/v24.17.0 or CVE-2026-4861x/4862x. Fix is the same low-risk pattern already validated in prior proposals (base-image bump only, no app code changes), and it only touches the `admins` tenant so it carries no `labs` memory risk.
**Source:** inbox "Source: cve" — Node.js June 2026 security releases (v22.23.0 / v24.17.0 / v26.3.1, 2026-06-18). https://nodejs.org/en/blog/vulnerability/june-2026-security-releases

### 2. backstage-v1521-kubernetes-config-fix: Adopt Backstage v1.52.1 for the kubernetes plugin config-schema correction
**Category:** refactor
**Impact:** 3
**Effort:** 2
**Rationale:** v1.52.1 is a patch release (not the major/next-day-of-release bump ADR 0001 tells us to avoid) that fixes a config-schema bug in the `kubernetes` plugin we actively use for the pods/services/CRDs viewer, plus an app sign-in runtime-init bug and a scheduler NULL bug. This compounds with our known footgun (`k8s-reader stale token` in architecture.md) — a broken kubernetes-plugin schema is exactly the kind of thing that turns a token-rotation event into a longer outage. No overlap with the existing `backstage-*` proposals in `proposals/`, all of which target the older v1.50.4 security patch.
**Source:** inbox "Source: github releases" — Backstage v1.52.1 (June 26, 2026).

### 3. version-reconciliation-check: Reconcile mctl-portal's reported version sources (package.json vs ArgoCD revision) and stabilize mctl service-config lookups
**Category:** feature
**Impact:** 2
