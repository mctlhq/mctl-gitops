## Rollback

This change is contained entirely within two frontend source files
(`web/src/tg.ts` and `web/src/screens/OrderDetail.tsx`). Rolling back requires
reverting those two files to their state on `main` before this branch was merged.

No database migration, no environment variable change, and no backend code is
involved, so rollback has zero infrastructure impact. The `public/` build artefact
is regenerated by CI on every deploy; reverting the source and re-deploying is
sufficient.

Steps:
1. `git revert <merge-commit-sha>` on `main` (or revert the feature branch commits
   individually if the merge was a merge commit).
2. Merge the revert PR through normal CI and review.
3. The next deployment will serve the pre-confirmation build.

There is no data to migrate back; user-facing behaviour reverts to the
no-confirmation state that existed before this change.
