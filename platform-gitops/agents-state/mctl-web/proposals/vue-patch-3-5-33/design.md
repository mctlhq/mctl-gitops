# Design: vue-patch-3-5-33

## Текущее состояние
Согласно `context/architecture.md`, mctl-web использует Vue 3.5.30 как часть стека Nuxt 4.3.1. Vue является транзитивной зависимостью Nuxt, но также может быть явно объявлен в `package.json` как peer dependency для обеспечения точной версии. Три страницы (`/`, `/docs`, `/privacy`) используют Vue-компоненты; Cloudflare Worker не использует Vue напрямую.

## Предлагаемое решение
Минимальное изменение: обновить запись `"vue"` в `package.json` с `^3.5.30` (или текущего значения) на `^3.5.33`. Запустить `npm install` для обновления lock-файла.

Если Vue не объявлен явно в `package.json` (управляется только как транзитивная зависимость через Nuxt), то обновить lock-файл командой `npm update vue@3.5.33`.

Никаких изменений в исходном коде не требуется: patch-версии Vue не вносят breaking changes по политике semver.

**Порядок относительно других предложений:**
Это обновление следует выполнить до `nuxt-upgrade-4-4-2`, чтобы изолировать изменения и упростить отладку.

## Альтернативы

### 1. Пропустить патч-обновление Vue и обновить сразу в рамках nuxt-upgrade-4-4-2
Допустимо технически, но усложняет изоляцию проблем. Если `nuxt-upgrade-4-4-2` введёт регрессию, неясно, из-за Vue или из-за Nuxt/vue-router. Раздельные PR упрощают rollback.

### 2. Не обновлять до явного выхода security advisory для Vue 3.5.30
Отклонено: патч-обновление — стандартная практика; откладывание создаёт накопленный технический долг и усложняет будущие обновления.

## Влияние на платформу

### Migration/миграции
Нет. Только изменение версии в `package.json` и lock-файла.

### Backward compatibility
Полная. Patch-версии Vue гарантируют отсутствие breaking changes.

### Resource impact
Отсутствует. Не затрагивает ни тенант admins, ни тенант labs. Bundle size практически не меняется в рамках patch-версий.

### Риски и митигации
| Риск | Вероятность | Митигация |
|---|---|---|
| Patch-версия содержит неожиданный regression | Крайне низкая | `nuxt build` + smoke-тест до merge |
| Lock-файл конфликтует с другими PR | Низкая | Координация с другими open PR, rebase при необходимости |
