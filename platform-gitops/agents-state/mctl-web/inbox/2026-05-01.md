## Top-3 (for spec-writer)

### 1. wrangler-ci-injection-audit: Audit CI/CD pipeline for CVE-2026-0933 wrangler command injection
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-0933 (CVSS 7.7) affects wrangler >= 4.0.0 < 4.59.1 via unsanitized `--commit-hash` in shell; the current bundled version 4.87.0 is above the patched threshold, but mctl-web has an exceptional self-contained `deploy.yml` that sources the commit hash from GitHub Actions context — this pipeline must be audited to confirm no untrusted external input flows into the `--commit-hash` argument. A confirmed-clean audit eliminates a high-severity CI/CD attack vector unique to this service.
**Source:** https://github.com/advisories/GHSA-36p8-mvp6-cv38

### 2. nuxt-upgrade-4-4: Upgrade Nuxt from 4.3.1 to 4.4.4
**Category:** performance
**Impact:** 3
**Effort:** 2
**Rationale:** Nuxt 4.4.x delivers module loading parallelization and Vite performance improvements that directly reduce cold-start and build times for mctl-web's SSG pipeline; bug fixes for async data and cookie serialization address patterns actively used in the Cloudflare Worker OAuth flow and tenant onboarding form. Staying current also keeps the service aligned with the Nuxt ecosystem security patch cadence, given that v4.x is the accepted ADR stack.
**Source:** https://github.com/nuxt/nuxt/releases/latest

### 3. vue-router-v5-migration: Migrate vue-router from 4.6.4 to 5.x
**Category:** refactor
**Impact:** 3
**Effort:** 2
**Rationale:** vue-router v4.x has ceased receiving new releases as of this reporting period, meaning future security patches and bug fixes will only land in v5; the v5 release has no breaking changes relative to v4 and merges unplugin-vue-router into core, reducing the dependency surface. Migrating now while the delta is small avoids a forced high-urgency migration later when v4 falls further behind.
**Source:** https://github.com/vuejs/router/releases/latest
