## Top-3 (for spec-writer)

### 1. wrangler-cve-command-injection: Confirm wrangler is past CVE-2026-0933 fix and harden CI pipeline
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** CVE-2026-0933 is an OS command injection in `wrangler pages deploy` triggered via the unsanitized `--commit-hash` parameter, affecting all wrangler versions up to 4.59.0. The service deploys via a GitHub Actions workflow that calls wrangler directly, making this an active attack surface if the pinned version falls in the vulnerable range. The fix was shipped in 4.59.1 and the current latest (4.88.0) is safe, but the in-use version is unconfirmed — the immediate action is to audit and pin wrangler >= 4.59.1 in `deploy.yml`, and add a step to fail the build if the resolved version is vulnerable.
**Source:** https://github.com/advisories/GHSA-36p8-mvp6-cv38

### 2. nuxt-minor-upgrade: Upgrade Nuxt from 4.3.1 to 4.4.3
**Category:** performance
**Impact:** 3
**Effort:** 2
**Rationale:** Nuxt 4.4.3 (released 2026-04-29) is two minor versions ahead of the current 4.3.1 and likely includes bug fixes, performance improvements, and compatibility updates with the Vue 3.5.x patch series also recently updated. Nuxt is the core SSG framework for the service, so staying current reduces the risk of accumulating hard-to-upgrade gaps. Vue 3.5.30 to 3.5.34 patch fixes are naturally pulled in as part of this upgrade.
**Source:** https://github.com/nuxt/nuxt/releases/tag/v4.4.3

### 3. vue-router-v5-migration: Evaluate and plan migration to vue-router v5
**Category:** feature
**Impact:** 3
**Effort:** 3
**Rationale:** Vue Router v5.0.6 integrates `unplugin-vue-router` into core, providing typed file-based routing natively — a meaningful DX improvement for the Nuxt 4 SSG setup where routes are already file-based. The upstream claims no breaking changes from v4, making this a lower-risk major bump than typical. A proposal should confirm compatibility with Nuxt 4.4.x and the existing vee-validate/yup forms before committing, but the payoff in type safety and reduced manual route declarations justifies scoping the work now.
**Source:** https://github.com/vuejs/router/releases/tag/v5.0.6
