## Top-3 (for spec-writer)

### 1. nuxt-upgrade-4-4-4: Upgrade Nuxt to v4.4.4
**Category:** performance
**Impact:** 4
**Effort:** 1
**Rationale:** Nuxt v4.4.4 shipped on 2026-04-29 (re-published from v4.4.3 due to a release-script issue) with meaningful performance improvements — TypeScript path resolution caching, module loading parallelization, and Nitro import optimizations — plus bug fixes for cookie serialization and async data handling. Production is at 4.3.1; the existing `nuxt-upgrade-4-4-2` proposal now targets a stale point release, and this proposal supersedes it with the correct target. Upgrading closes a growing gap (4.3.1 → 4.4.4) before it widens further.
**Source:** https://github.com/nuxt/nuxt/releases/latest — nuxt v4.4.4, released 2026-04-29.

### 2. workerd-runtime-upgrade: Upgrade workerd runtime to v1.20260430.1
**Category:** security
**Impact:** 3
**Effort:** 1
**Rationale:** workerd v1.20260430.1 (released 2026-04-30) fixes a use-after-free bug with captured `this` values — a memory-safety defect in the Worker runtime that processes all `/api/*` traffic (OAuth callbacks, tenant provisioning, contact forms). It also upgrades the Node.js layer to v22 and adds sidecar container state improvements. The existing `workerd-clearweak-migration` proposal addresses a code-level API deprecation; this proposal is distinct — it targets the runtime binary upgrade for the stability/safety fix.
**Source:** https://github.com/cloudflare/workerd/releases/latest — workerd v1.20260430.1, released 2026-04-30.

### 3. worker-fetch-observability: Add structured error handling to the Cloudflare Worker
**Category:** feature
**Impact:** 3
**Effort:** 2
**Rationale:** workerd v1.20260430.1 introduces improved fetch error messaging, surfacing richer diagnostics via `wrangler tail`. The `/api/*` Worker currently has no structured error-response contract — errors from Backstage, GitHub OAuth, Resend, or Telegram are likely swallowed or forwarded as opaque strings. Adding a thin error-handling middleware layer now (1) takes advantage of the new runtime diagnostics, (2) gives the frontend consistent JSON error shapes to display, and (3) shortens mean time to diagnose incidents. Not blocked by any ADR.
**Source:** https://github.com/cloudflare/workerd/releases/latest — workerd v1.20260430.1 "improved fetch error messaging".
