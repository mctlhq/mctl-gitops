## Top-3 (for spec-writer)

### 1. Patch @cloudflare/vite-plugin to fix secret-exposure CVE
- **Slug:** cloudflare-vite-plugin-cve-fix
- **Category:** security
- **Impact:** 5
- **Effort:** 1
- **Rationale:** CVE-2025-59427 allows any local network actor to read `.env` and `.dev.vars` through the Vite dev server, which in mctl-web's case contains GITHUB_CLIENT_SECRET, TELEGRAM_BOT_TOKEN, RESEND_API_KEY, and BACKSTAGE_LANDING_TOKEN. These secrets are used in production OAuth and tenant-provisioning flows, so exposure would be catastrophic. Upgrading @cloudflare/vite-plugin to >= 1.6.0 is a one-line version bump with a published fix.
- **Source:** CVE-2025-59427 / GHSA-4pfg-2mw5-f8jx; fixed in @cloudflare/vite-plugin 1.6.0

### 2. Pin CI Node.js to >= 22 before upgrading wrangler to 4.87.0
- **Slug:** wrangler-node22-ci-upgrade
- **Category:** security
- **Impact:** 4
- **Effort:** 2
- **Rationale:** wrangler @4.87.0 hard-fails on Node.js < 22, and mctl-web's `deploy.yml` is the only in-repo deploy pipeline on the platform — a silent Node version mismatch would break all Worker deploys and halt the production release cycle. Pinning the GitHub Actions `node-version` to 22 and bumping wrangler together removes the risk before it manifests in CI. This is a CI/CD blocking change, not just a nice-to-have version bump.
- **Source:** cloudflare/workers-sdk wrangler @4.87.0 release notes (2026-04-30)

### 3. Upgrade Nuxt 4.3.1 → 4.4.4 for FormData deduplication fix
- **Slug:** nuxt-444-formdata-fix
- **Category:** performance
- **Impact:** 3
- **Effort:** 2
- **Rationale:** The `/api/submit` endpoint receives FormData for tenant provisioning and is subject to Nuxt's `useFetch` deduplication logic; the body-hashing bug in 4.3.x can cause duplicate or dropped requests on retry, silently failing tenant creation. The 4.4.4 patch also fixes manifest retry logic which affects prerendered page hydration. This is a targeted patch-level upgrade with no breaking changes and a direct functional fix for a core mctl-web user journey.
- **Source:** nuxt/nuxt v4.4.4 release (2026-04-29)
