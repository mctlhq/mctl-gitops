## Top-3 (for spec-writer)

### 1. workerd-json-crash-fix: Pin wrangler to pull workerd v1.20260505.1 crash fix
**Category:** performance
**Impact:** 3
**Effort:** 1
**Rationale:** workerd v1.20260505.1 shipped today with an explicit crash fix for JSON module handling in Workers. The mctl-web Cloudflare Worker may serve and process JSON data; a crash path in the runtime is a direct stability risk. Since workerd is a transitive dependency of wrangler, a single wrangler version bump (already recommended in `wrangler-cve-2026-0933`) needs to be re-verified to confirm it now also pulls in v1.20260505.1 — this follow-on proposal ensures the crash fix is explicitly tracked. No existing proposal targets this specific fix.
**Source:** https://github.com/cloudflare/workerd/releases/latest

### 2. automated-dep-updates: Introduce Renovate for automated dependency tracking
**Category:** refactor
**Impact:** 4
**Effort:** 2
**Rationale:** workerd releases daily (v1.20260504.1 → v1.20260505.1 within 24 hours); wrangler, Nuxt, and Vue are on rapid release cadences too. Manual researcher tracking cannot keep pace with security fixes appearing between daily cycles, as demonstrated by the workerd crash fix landing the day after the previous inbox was filed. Renovate (or Dependabot) configured for the mctl-web repo would raise PRs automatically, shrink mean-time-to-patch, and let the researcher focus on higher-signal analysis rather than version enumeration. No existing ADR or proposal covers this.
**Source:** workerd v1.20260505.1 vs v1.20260504.1 (24-hour gap); wrangler CVE-2026-0933 (patched versions existed days before being caught manually)

### 3. vueuse-nuxt-autoimports: Enable vueuse Nuxt auto-import composables
**Category:** feature
**Impact:** 2
**Effort:** 1
**Rationale:** vueuse 14.3.0 (already in the pending `vueuse-upgrade-14-3` proposal) ships Nuxt composable auto-import variants, meaning composables like `useStorage`, `useLocalStorage`, and `useDark` can be used in Nuxt pages without explicit import statements. Enabling this via `nuxt.config.ts` reduces boilerplate in every page component and aligns with Nuxt 4's auto-import philosophy already used for Vue/Nuxt built-ins. This is a configuration step distinct from the version upgrade itself; no existing proposal covers the Nuxt auto-import integration.
**Source:** https://github.com/vueuse/vueuse/releases/latest
