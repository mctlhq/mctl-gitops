## Top-3 (for spec-writer)

### 1. vueuse-upgrade-14-3: Upgrade @vueuse/core from 14.2.1 to 14.3.0
**Category:** performance
**Impact:** 2
**Effort:** 1
**Rationale:** mctl-web directly depends on @vueuse/core 14.2.1 (confirmed in architecture.md) and 14.3.0 ships bug fixes in useWebSocket and useWakeLock that could affect runtime reliability on any page using those composables. The upgrade is a minor semver bump with no breaking changes, making effort minimal. This is the only genuinely novel signal today that is not covered by an existing proposal.
**Source:** https://github.com/vueuse/vueuse/releases/latest

### 2. sass-upgrade-1-99: Upgrade sass from 1.98.0 to 1.99.0
**Category:** refactor
**Impact:** 2
**Effort:** 1
**Rationale:** sass 1.99.0 introduces native support for the parent selector (`&`) at document root level, emitting scoping-root selectors directly into CSS output; this is a build-time dependency used by mctl-web and staying current avoids accumulating version debt. The existing `sass-deprecation-fix` proposal targets deprecation-warning suppressions and does not cover this version increment. The upgrade carries no runtime risk as sass is a build-only tool.
**Source:** https://github.com/sass/dart-sass/releases/latest

### 3. wrangler-4-87-upgrade: Upgrade Wrangler to 4.87.0 for module-fallback V2 support
**Category:** refactor
**Impact:** 2
**Effort:** 1
**Rationale:** Wrangler 4.87.0 adds V2 protocol support in the module fallback service and refactors the router-worker for gradual rollout gating; both improvements reduce the risk of silent Worker mis-routing during deployments. All existing wrangler proposals (wrangler-cve-0933, wrangler-upgrade-security, wrangler-full-upgrade, wrangler-ci-injection-audit) target the CVE-2026-0933 security fix and do not address this functional release. Keeping the CI deploy tool current also reduces supply-chain drift.
**Source:** https://github.com/cloudflare/workers-sdk/releases/latest
