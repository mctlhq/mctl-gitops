## Source: cve

- **CVE-2026-47200 — Nuxt route middleware bypass via component islands** — When component islands are enabled, `.server.vue` pages are reachable via `/__nuxt_island/page_*` endpoints that render without invoking Vue Router, bypassing middleware-based access control entirely; unauthenticated attackers can hit these endpoints directly. Fixed in nuxt 3.21.6 / 4.4.6. [Corgea](https://corgea.com/advisories/vulnerabilities/CVE-2026-47200)

- **CVE-2026-46342 — Nuxt island response validation issue** — Affects Nuxt 3.1.0–3.21.5 and 4.0.0-alpha.1–4.4.5 (mctl-web's architecture doc pins Nuxt at 4.3.1, inside the affected range). [vuldb](https://vuldb.com/cve/CVE-2026-45670) / related coverage via Nuxt CVE roundup search.

- **CVE-2026-45670 — Nuxt dev server source code exposure** — Running `nuxt dev --host` binds the dev server to a non-loopback address; with the rspack/webpack builder, other hosts on the same network can access application source code. [vuldb](https://vuldb.com/cve/CVE-2026-45670)

- **CVE-2026-45669 — Nuxt reflected XSS in `navigateTo()`** — When `navigateTo()` is called with `external: true`, Nuxt emits a server-rendered HTML meta-refresh redirect; the destination URL is insufficiently sanitized (HTML-significant characters not encoded), allowing script injection if an attacker controls the URL parameter. Fixed together with the above in nuxt 3.21.6 / 4.4.6. [Netlify security changelog](https://www.netlify.com/changelog/2026-05-19-nuxt-security-vulnerabilities/)

- **GHSA-8c93-4hch-xgxp — Cloudflare Wrangler directory traversal vulnerability** — New advisory surfaced in this search pass (not seen in the 2026-05-16 cycle) for `cloudflare/workers-sdk`; distinct from the previously-reported CVE-2026-0933 command-injection advisory. ## Top-3 (for spec-writer)

### 1. nuxt-cve-upgrade-4-4-8: Upgrade Nuxt to 4.4.8 to close three CVEs (island middleware bypass, island response validation, reflected XSS)
**Category:** security
**Impact:** 5
**Effort:** 2
**Rationale:** Three distinct CVEs affect our pinned Nuxt 4.3.1: CVE-2026-47200 (component-island endpoints bypass Vue Router middleware entirely, allowing unauthenticated access to routes meant to be gated), CVE-2026-46342 (island response validation, affecting the 4.0.0-alpha.1–4.4.5 range which includes our pin), and CVE-2026-45669 (reflected XSS via `navigateTo({external:true})`). All three are fixed by 4.4.6+, and the latest available release (4.4.8) is a low-risk hotfix on top of that. This is a pure version bump — it does not revert Nuxt or touch vee-validate/yup/the Worker, so it is fully compatible with ADR 0001. Given mctl-web is a public site with GitHub OAuth and tenant-provisioning forms, closing a middleware-bypass + XSS combination is the single highest-impact item this cycle.
**Source:** nuxt/nuxt v4.4.8 (https://github.com/nuxt/nuxt/releases/latest); CVE-2026-47200; CVE-2026-46342; CVE-2026-45669

### 2. mctl-web-status-mismatch: Investigate and reconcile version/host mismatch between live service status and context docs
**Category:** refactor
**Impact:** 4
**Effort:** 2
**Rationale:** `mctl_get_service_status` reports `imageTag: 7.1.1` and `host: mctl.me` for `admins/mctl-web`, while `context/current-version.md` says version `4.6.2` (last updated 2026-04-25) and `context/architecture.md` names `mctl.ai` as the primary domain with `mctl.me` only as a redirect source. This is either a serious documentation-drift problem (the version-bump process described in `current-version.md` — "update this file and add an ADR" — is clearly not being followed, which undermines every other proposal that relies on `context/` as ground truth) or a real deploy/routing misconfiguration (production traffic answering on the wrong host). Either way it is cheap to investigate and directly affects the reliability of this very analysis pipeline.
**Source:** mctl_get_service_status (tenant admins, service mctl-web); context/current-version.md; context/architecture.md

### 3. wrangler-traversal-ghsa-8c93: Assess Cloudflare Wrangler directory-traversal advisory (GHSA-8c93-4hch-xgxp) against our deploy pipeline
**Category:** security
**Impact:** 3
**Effort:** 3
**Rationale:** A new directory-traversal advisory against `cloudflare/workers-sdk` (Wrangler) surfaced this cycle, distinct from the already-tracked CVE-2026-0933 command-injection issue. Wrangler is architecturally in-scope: `architecture.md` states the Worker is deployed "via Wrangler via GitHub Actions" as the one exception to centralized mctl-gitops builds, so a Wrangler-tooling vulnerability sits directly in our CI/CD path. The researcher could not extract affected/fixed version ranges this pass, so this item's first task is a follow-up read of the advisory before a concrete patch can be scoped — kept in the Top-3 despite the information gap because of where it sits (build/deploy tooling with repo/secrets access) rather than because impact is already confirmed.
**Source:** GHSA-8c93-4hch-xgxp (https://github.com/cloudflare/workers-sdk/security/advisories/GHSA-8c93-4hch-xgxp)
