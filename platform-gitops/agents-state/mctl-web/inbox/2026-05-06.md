## Top-3 (for spec-writer)

### 1. wrangler-4-88-upgrade: Wrangler v4.88.0 upgrade with stabilized secrets config
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** wrangler v4.88.0 (released 2026-05-05) is one version ahead of the highest existing proposal (`wrangler-4-87-upgrade` targets 4.87.0). The stabilization of the `secrets` config property means secrets management in `wrangler.toml` graduates from experimental to stable API, directly reducing the risk of misconfigurations that could leak secrets bound to the `/api/*` Worker endpoints. Keeping the CI deploy tool on the latest stable version also maintains the project's exact-pin auditability policy established by prior wrangler proposals.
**Source:** https://github.com/cloudflare/workers-sdk/releases/latest — wrangler v4.88.0, released 2026-05-05.

### 2. workerd-v8-148-upgrade: workerd v1.20260506.1 with V8 14.8
**Category:** performance
**Impact:** 3
**Effort:** 2
**Rationale:** workerd v1.20260506.1 released today ships V8 14.8, which typically includes JIT improvements and memory layout optimizations that reduce CPU time for the JavaScript executed in the `/api/*` Worker handlers. The existing `workerd-runtime-upgrade` proposal targets v1.20260430.1 (the version bundled with wrangler 4.86.0); today's release is a full week newer and is the version that will be bundled with wrangler 4.88.0. Upgrading wrangler to 4.88.0 (finding #1) implicitly delivers this workerd update, but an explicit pin in `cloudflare-worker/package.json` ensures local `wrangler dev` matches production. No Kubernetes tenant memory impact — the Worker runs on Cloudflare infrastructure.
**Source:** https://github.com/cloudflare/workerd/releases/latest — workerd v1.20260506.1, released 2026-05-06.

### 3. wrangler-interactive-prompt-ci-guard: Guard CI against wrangler 4.88 interactive prompt hangs
**Category:** feature
**Impact:** 3
**Effort:** 2
**Rationale:** wrangler v4.88.0 introduces interactive prompts that fire when the Worker name or `compatibility_date` is absent from `wrangler.toml`. In a non-interactive CI environment (GitHub Actions `deploy.yml`), an unanswered prompt causes the pipeline to hang indefinitely until the job times out, which would block all production deployments. The fix requires auditing `wrangler.toml` and `deploy.yml` to ensure `name` and `compatibility_date` are always explicitly set, and optionally passing `--no-interactive` (or the equivalent env flag `CI=true`) to the wrangler invocation. This is a low-effort, high-reliability guard that must be applied alongside finding #1.
**Source:** https://github.com/cloudflare/workers-sdk/releases/latest — wrangler v4.88.0 changelog, interactive prompts feature, released 2026-05-05.
