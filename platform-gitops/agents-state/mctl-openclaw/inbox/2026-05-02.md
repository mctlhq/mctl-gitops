## Top-3 (for spec-writer)

### 1. cve-addendum-auth-bypass-exec-bypass: Three new OpenClaw CVEs affecting current version (auth bypass, webhook replay, exec allowlist bypass)
**Category:** security
**Impact:** 4
**Effort:** 1
**Rationale:** CVE-2026-41394 (plugin-auth route auth bypass allowing unauthenticated writes), CVE-2026-41395 (Plivo V3 webhook replay bypass), and CVE-2026-41390 (exec allowlist bypass via shell wrappers) were published after yesterday's inbox and are not covered by any existing proposal. All three affect our current deployment version 2026.3.14. They reinforce and extend the urgency of the existing `upgrade-to-2026-4-29` proposal and require explicit CVE tracking for compliance/audit traceability — the same pattern used in `cve-addendum-ssrf-eval-bypass`. CVE-2026-41394 in particular (unauthenticated access to operator-write scopes) is a high-severity runtime security risk on all three tenants. All are fixed in openclaw 2026.4.29.
**Source:** https://www.redpacketsecurity.com/cve-alert-cve-2026-41394-openclaw-openclaw/ ; search result summary for CVE-2026-41395, CVE-2026-41390

### 2. discord-supply-chain-typosquat: Malicious `discord.js-user` npm package exfiltrates Discord tokens
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** GHSA-69r6-7h4f-9p7q identifies `discord.js-user` as a malicious npm package that silently uploads Discord bot tokens to an attacker-controlled server. Any transitive dependency in the openclaw workspace that accidentally resolves to this typosquat (instead of `discord.js`) would compromise all Discord channel bot tokens across all three tenants. The existing `baileys-registry-lockdown` proposal establishes the pattern for registry lockdowns on channel libraries; this proposal extends that pattern to Discord. The impact is scoped to `labs`, `admins`, and `ovk` Discord channels and their associated bot tokens. Effort is low because the technical approach (explicit package version pinning + `.npmrc` deny-list entry) mirrors the Baileys lockdown.
**Source:** https://www.miggo.io/vulnerability-database/cve/GHSA-69r6-7h4f-9p7q

### 3. discordjs-v14264-update: Upgrade discord.js to v14.26.4 (fixes DM reception in uncached channels)
**Category:** feature
**Impact:** 2
**Effort:** 1
**Rationale:** discord.js v14.26.4 patches a regression where bots fail to receive direct messages when the target DMChannel is not yet in cache — a scenario that is common on startup and after reconnects. Discord is an active channel in openclaw across all three tenants; silently dropped DMs reduce perceived reliability for users. This fix may be bundled into the openclaw 2026.4.29 upgrade already proposed, so the primary task in this proposal is verification: confirm the 2026.4.29 upstream package.json includes discord.js ≥ 14.26.4, and if not, raise a targeted patch PR to the openclaw fork's Discord extension. Effort is 1 because the action is either a one-line version bump or a verification-only task. Not flagged as risky for `labs` — the discord.js update introduces no new dependencies.
**Source:** https://github.com/discordjs/discord.js/releases (v14.26.4)
