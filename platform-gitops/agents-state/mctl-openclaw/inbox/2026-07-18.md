### 1. baileys-ghsa-qvv5-patch: Upgrade @whiskeysockets/baileys to patch GHSA-qvv5-jq5g-4cgg
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** A real security advisory in the legitimate `@whiskeysockets/baileys` package itself (not a typosquat) was patched in v7.0.0-rc12, with a follow-up regression fix in rc13 (protocolMessage/`fromMe` parsing). This is distinct from the existing `baileys-supply-chain-audit` / `baileys-registry-lockdown` / `baileys-lockfile-audit` proposals, which all address the `lotusbail` typosquat and registry pinning, not an upstream CVE in the genuine package — so this does not duplicate accepted work. Baileys powers the WhatsApp channel across all three tenants (architectural fit), version bumps in this line have historically been memory-neutral or memory-reducing (rc10 shipped leak fixes), so this is low-risk for the `labs` memory limit.
**Source:** inbox 2026-07-18, "Source: github releases" — WhiskeySockets/Baileys v7.0.0-rc12/rc13; "Source: cve" — GHSA-qvv5-jq5g-4cgg.

### 2. nodejs-june2026-cve-patch: Patch Node.js base image for the June 2026 security release (2 High CVEs)
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** The June 18, 2026 Node.js security release fixes two High-severity CVEs relevant to a network-facing gateway: CVE-2026-48933 (WebCrypto integer-overflow remote DoS/crash) and CVE-2026-48618 (TLS hostname-verification bypass breaking wildcard cert trust), plus HTTP/2 memory-exhaustion and proxy-credential-disclosure issues. Prior proposals (`nodejs-security-patch`, `nodejs-runtime-upgrade`) target the March and January 2026 releases respectively — neither covers this June release, so this is new, non-duplicate ground. A crash-inducing DoS CVE directly threatens `ovk`'s SLA (pod restart before the restore-state probe passes), and a base-image patch bump follows the same low-risk, no-RSS-impact pattern already validated in `nodejs-security-patch`. Also verify none of the three tenants' images are still on Node 20 (EOL, no vendor patch path) as part of this work.
**Source:** inbox 2026-07-18, "Source: cve" — Node.js June 18, 2026 security releases; Node 20 EOL warning.
