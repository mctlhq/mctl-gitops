### 1. openclaw-cve-upgrade: Upgrade openclaw from 2026.3.14 to 2026.5.6 to remediate critical CVEs
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** We are currently running 2026.3.14 and are exposed to at least seven confirmed CVEs against our stack, including two rated CVSS 9.8 Critical (CVE-2026-43575 noVNC Auth Bypass, CVE-2026-44109 Feishu Webhook RCE), one CVSS 9.6 Critical (CVE-2026-43581 Chrome DevTools Protocol exposed on 0.0.0.0), two CVSS 8.8/8.6 High (CVE-2026-44115 shell expansion allowlist bypass, CVE-2026-44116 Zalo SSRF), plus SSRF policy bypass (CVE-2026-43573, CVSS 7.7) and Slack prompt injection (CVE-2026-33579). The minimum version to close all of these is 2026.4.22; upgrading to the latest stable 2026.5.6 closes all known CVEs and also picks up 120+ upstream fixes. The rollout must follow the labs -> admins -> ovk pipeline per ADR-0001, with S3 canary stopped and restore-state probe respected per ADR-0002; note that `labs` is near its memory limit so any measurable RSS increase from the upgrade must be measured and documented before promoting to `ovk`.

### 2. nodejs-security-patch: Update Node.js runtime to patch two high-severity process-crash CVEs
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-21637 (TLS SNICallback uncaught exception crashes the process) and CVE-2026-21710 (`__proto__` HTTP header triggers uncaught TypeError and crashes the process) both affect all Node.js LTS lines we could be running (20.x, 22.x, 24.x). A process crash in a Kubernetes pod triggers an unplanned restart which, before the restore-state probe completes, means channel sessions are temporarily unavailable; for `ovk` this is a direct SLA risk. Patching requires bumping the Node.js base image in the Dockerfile to the patched LTS release (22.22.2, 20.20.2, or 24.x equivalent); no application code changes are expected and RAM impact is negligible.
**Source:** CVE-2026-21637 https://nodejs.org/en/blog/vulnerability/march-2026-security-releases | CVE-2026-21710 https://nodejs.org/en/blog/vulnerability/march-2026-security-releases | Node.js 22.22.2 Security Release 2026-03-24

### 3. typescript-6-migration: Migrate workspace to TypeScript 6.0 ahead of breaking defaults
**Category:** refactor
**Impact:** 2
**Effort:** 3
**Rationale:** TypeScript 6.0.3 is now stable and ships with breaking default changes: `strict` is on by default, `module` defaults to `esnext`, `target` defaults to `es2025`, `types` defaults to `[]`, and `--moduleResolution node` is deprecated. Our workspace packages currently rely on explicit or inherited tsconfig settings that may have been authored against TS 5.x defaults; without a controlled migration the team will hit unexpected type errors when developers update their local toolchains or when CI tooling upgrades. The effort is moderate — it requires an audit of all `tsconfig.json` files in `extensions/*` and core, explicit opt-in or opt-out for each changed default, and fixing any newly surfaced type errors — but there is no memory or runtime impact, making it safe for `labs`.
**Source:** TypeScript 6.0.3 release https://github.com/microsoft/TypeScript/releases | stable 2026-04-16
