## Top-3 (for spec-writer)

### 1. upgrade-to-2026-4-26: Upgrade openclaw to 2026.4.26 (supersedes 2026.4.25 target)
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** The currently deployed version 2026.3.14 is exposed to six HIGH CVEs (CVE-2026-41371, -41349, -41342, -41352, -41353, -41359) covering privilege escalation, agentic consent bypass, auth bypass, and RCE-class node-pairing flaws — all fixed in 2026.3.22–2026.3.31. Today's 2026.4.26 release additionally closes bearer-token echo in `device.token.rotate`, making it the correct upgrade target rather than yesterday's 2026.4.25. The existing `upgrade-to-2026-4-25` proposal must be superseded by this new target version; the slug is intentionally distinct.
**Source:** https://github.com/openclaw/openclaw/releases; https://cve.threatint.eu/CVE/CVE-2026-41371; https://ccb.belgium.be/advisories/warning-3-high-severity-vulnerabilities-openclaw-can-lead-rce-patch-immediately

### 2. axios-supply-chain-pinning: Audit and pin Axios transitive dependencies against WAVESHAPER.V2 backdoor
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** UNC1069 (North Korea-linked) backdoored Axios 1.14.1 and 0.30.4 with the WAVESHAPER.V2 RAT, which steals credentials from browsers, npm tokens, and GitHub tokens — exactly the kind of secrets our deployment holds in S3 and env vars. While node-slack-sdk is confirmed safe at axios@1.15.0, the full transitive dependency graph across all workspace packages has not been audited for the affected versions. This is a distinct, targeted threat from the general `npm-supply-chain-audit` proposal (which covered `lotusbail` and `discord.js-user` scanning only).
**Source:** https://thehackernews.com/2026/04/unc1069-social-engineering-of-axios.html

### 3. gateway-ui-responsiveness-fix: Track and patch upstream #73836 (Gateway/UI responsiveness regression + heartbeat noise)
**Category:** performance
**Impact:** 3
**Effort:** 2
**Rationale:** Upstream bug #73836 causes Telegram typing gaps and UI reconnect stalls on channels we actively use in production (`ovk`). More critically, the heartbeat poll noise it introduces risks triggering false s3-sync canary alerts — a footgun explicitly called out in ADR-0002 as a source of operational incidents. Monitoring the upstream fix and cherry-picking it to our fork prevents canary noise from masking real S3-sync failures.
**Source:** https://github.com/openclaw/openclaw/issues?labels=bug,security&state=open (issue #73836)
