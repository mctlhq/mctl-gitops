## Top-3 (for spec-writer)

### 1. openclaw-security-upgrade: Upgrade openclaw to 2026.4.25 — close critical CVEs
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** The current 2026.3.14 deployment is vulnerable to two critical (CVSS 8.7 and 8.4) and two high (CVSS 7.1) CVEs from the April batch, plus three more CVEs from the 2026.4134x series — totalling 4+ critical/high vectors including privilege escalation to operator, arbitrary code execution via .npmrc, session isolation bypass and path traversal in the media sandbox. All are closed by upgrading to >=2026.4.5; upgrading to 2026.4.25 (latest stable) closes the entire known batch in one move and removes the 6+ release lag. The labs → admins → ovk rollout matches ADR 0001; the restore-state probe (ADR 0002) guarantees ovk safety during deployment.
**Source:** CVE-2026-35639 (CVSS 8.7), CVE-2026-35641 (CVSS 8.4), CVE-2026-35636, CVE-2026-35668, CVE-2026-41297, CVE-2026-41342, CVE-2026-41359 — https://blink.new/blog/openclaw-april-2026-new-cves-security-patch-guide; https://github.com/openclaw/openclaw/releases

### 2. egress-network-policy: Restrict egress from openclaw pods to mitigate SSRF
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** CVE-2026-41297 — SSRF in marketplace plugin download — allows redirecting requests to arbitrary hosts inside the cluster and outside; even after upgrading openclaw the vector itself (the absence of an egress NetworkPolicy on the openclaw namespace) remains an architectural gap. A Kubernetes NetworkPolicy that limits egress from the three tenant namespaces to a whitelist (S3, upstream marketplace, channels) closes this attack class regardless of future SSRF in openclaw or its plugins. Effort is low: a NetworkPolicy manifest with no code changes, applied via ArgoCD GitOps; for labs RAM does not grow.
**Source:** CVE-2026-41297 — https://www.redpacketsecurity.com/cve-alert-cve-2026-41297-openclaw-openclaw/

### 3. nodejs-runtime-upgrade: Upgrade Node.js runtime to a safe version + audit lockfile for malicious packages
**Category:** security
**Impact:** 3
**Effort:** 2
**Rationale:** The Node.js January 2026 Security Release closes three High CVEs (CVE-2025-55131 buffer non-zerofilled, CVE-2025-55130 symlink bypass, CVE-2025-59465 HTTP/2 DoS) — if the current runtime image uses a version below v20.20.0/v22.22.0/v24.13.0, all three vectors are active. In parallel, an `npm audit` plus a lockfile grep for the `lotusbail` and `discord.js-user` packages is required (both are malicious supply-chain disclosures from late 2025): lotusbail exfiltrates WhatsApp auth tokens, discord.js-user exfiltrates Discord tokens, both can be present in transitive dependencies. Effort is minimal: base image bump in the Dockerfile + CI step `npm audit --audit-level=high`; RAM does not grow.
**Source:** Node.js January 2026 Security Release — https://nodejs.org/en/blog/vulnerability/december-2025-security-releases; lotusbail — https://www.bleepingcomputer.com/news/security/malicious-npm-package-steals-whatsapp-accounts-and-messages/; GHSA-69r6-7h4f-9p7q — https://www.miggo.io/vulnerability-database/cve/GHSA-69r6-7h4f-9p7q
