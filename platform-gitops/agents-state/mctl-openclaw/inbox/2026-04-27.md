## Top-3 (для spec-writer)

### 1. openclaw-security-upgrade: Обновление openclaw до 2026.4.25 — закрытие критических CVE
**Категория:** security
**Impact:** 5
**Effort:** 3
**Обоснование:** Текущий деплой 2026.3.14 уязвим к двум критическим (CVSS 8.7 и 8.4) и двум высоким (CVSS 7.1) CVE апрельского батча, плюс ещё трём CVE из серии 2026.4134x — итого 4+ critical/high векторов включая privilege escalation до operator, arbitrary code execution через .npmrc, session isolation bypass и path traversal в media sandbox. Все закрываются обновлением до >=2026.4.5; обновление до 2026.4.25 (latest stable) закрывает весь known batch разом и устраняет отставание в 6+ релизов. Rollout по схеме labs → admins → ovk соответствует ADR 0001; restore-state probe (ADR 0002) гарантирует безопасность ovk при деплое.
**Source:** CVE-2026-35639 (CVSS 8.7), CVE-2026-35641 (CVSS 8.4), CVE-2026-35636, CVE-2026-35668, CVE-2026-41297, CVE-2026-41342, CVE-2026-41359 — https://blink.new/blog/openclaw-april-2026-new-cves-security-patch-guide; https://github.com/openclaw/openclaw/releases

### 2. egress-network-policy: Ограничение egress из openclaw pods для митигации SSRF
**Категория:** security
**Impact:** 4
**Effort:** 2
**Обоснование:** CVE-2026-41297 — SSRF в marketplace plugin download — позволяет перенаправлять запросы на произвольные хосты внутри кластера и вовне; даже после обновления openclaw сам вектор (отсутствие egress NetworkPolicy на namespace openclaw) остаётся architectural gap. Kubernetes NetworkPolicy, ограничивающая egress из трёх тенантных namespace до белого списка (S3, upstream marketplace, каналы), закроет этот класс атак независимо от будущих SSRF в openclaw или его плагинах. Effort низкий: NetworkPolicy манифест без кодовых изменений, применяется через ArgoCD gitops; для labs не увеличивает RAM.
**Source:** CVE-2026-41297 — https://www.redpacketsecurity.com/cve-alert-cve-2026-41297-openclaw-openclaw/

### 3. nodejs-runtime-upgrade: Обновление Node.js runtime до безопасной версии + аудит lockfile на malicious packages
**Категория:** security
**Impact:** 3
**Effort:** 2
**Обоснование:** Node.js January 2026 Security Release закрывает три High CVE (CVE-2025-55131 buffer non-zerofilled, CVE-2025-55130 symlink bypass, CVE-2025-59465 HTTP/2 DoS) — если текущий runtime образ использует версию ниже v20.20.0/v22.22.0/v24.13.0, все три вектора активны. Параллельно необходим `npm audit` + grep lockfile на пакеты `lotusbail` и `discord.js-user` (оба — malicious supply-chain, раскрыты в конце 2025): lotusbail эксфильтрирует WhatsApp auth tokens, discord.js-user — Discord tokens, оба могут находиться в транзитивных зависимостях. Effort минимальный: base image bump в Dockerfile + CI-шаг `npm audit --audit-level=high`; RAM не увеличивается.
**Source:** Node.js January 2026 Security Release — https://nodejs.org/en/blog/vulnerability/december-2025-security-releases; lotusbail — https://www.bleepingcomputer.com/news/security/malicious-npm-package-steals-whatsapp-accounts-and-messages/; GHSA-69r6-7h4f-9p7q — https://www.miggo.io/vulnerability-database/cve/GHSA-69r6-7h4f-9p7q
