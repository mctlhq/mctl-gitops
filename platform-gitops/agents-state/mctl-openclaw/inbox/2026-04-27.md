## Top-3 (для spec-writer)

### 1. upgrade-to-2026-4-25: Обновление openclaw до 2026.4.25 (закрытие 5 незакрытых CVE)
**Категория:** security
**Impact:** 5
**Effort:** 3
**Обоснование:** Текущая версия 2026.3.14 содержит 5 незакрытых уязвимостей: CVE-2026-41349 (CVSS 8.8, agentic consent bypass), CVE-2026-41361 (SSRF IPv6 bypass), CVE-2026-41359 (Telegram priv esc), CVE-2026-41353 (allowProfiles bypass), CVE-2026-41348 (Discord auth bypass) — все закрыты в 2026.4.25. Наиболее критична CVE-2026-41349: LLM-агент может тихо отключить execution approval, что в agentic-среде openclaw означает неконтролируемое выполнение действий. Upgrade идёт по накатанному маршруту labs → admins → ovk согласно ADR 0001; RAM-footprint нужно проверить перед накатом на labs, но сам upgrade не добавляет новых компонентов принципиально.
**Source:** https://www.thehackerwire.com/vulnerability/CVE-2026-41349/ ; https://cve.threatint.eu/CVE/CVE-2026-41361 ; https://cve.threatint.eu/CVE/CVE-2026-41359 ; https://www.redpacketsecurity.com/cve-alert-cve-2026-41353-openclaw-openclaw/ ; https://www.vulncheck.com/advisories/openclaw-authorization-bypass-in-discord-text-approval-commands ; https://github.com/openclaw/openclaw/releases/latest

### 2. clawhub-skills-allowlist: Политика allowlist для external skills из ClawHub
**Категория:** security
**Impact:** 4
**Effort:** 2
**Обоснование:** Активная кампания ClawHavoc разместила 341+ вредоносных skills в официальном ClawHub marketplace; CVE не назначен, кампания продолжается. Платформа использует Layer 3 (remote skills, HTTP-delegated) — если хоть один тенант устанавливал skills с ClawHub без верификации, вектор уже открыт. Митигация требует фиксации allowlist разрешённых skill-источников в gitops-конфиге каждого тенанта и CI-проверки на появление новых неодобренных skills. Effort низкий: это конфигурационное изменение, не затрагивает RAM и не требует апстрим-патча.
**Source:** https://www.armosec.io/blog/cve-2026-32922-openclaw-privilege-escalation-cloud-security/ (ClawHavoc supply chain)

### 3. npm-supply-chain-audit: Аудит и пиннинг официальных npm-пакетов (Baileys, discord.js)
**Категория:** security
**Impact:** 3
**Effort:** 1
**Обоснование:** Зафиксированы активно распространяемые poisoned npm-форки: поддельный Baileys-форк перехватывает WhatsApp auth-токены и медиафайлы, `discord.js-user` (CVSS 9.8) сливает Discord-токены. Если в package-lock.json закреплены правильные пакеты (`@whiskeysockets/baileys`, `discord.js`), угроза минимальна — но явная проверка и CI-контроль (например, `npm audit` + проверка resolved URL на официальный реестр) устраняют риск случайной подмены при будущих обновлениях. Effort минимальный: одноразовый аудит + добавление CI-шага.
**Source:** https://www.theregister.com/2025/12/22/whatsapp_npm_package_message_steal/ ; https://www.miggo.io/vulnerability-database/cve/GHSA-69r6-7h4f-9p7q
