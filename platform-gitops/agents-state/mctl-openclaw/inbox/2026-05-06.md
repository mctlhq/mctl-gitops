## Top-3 (for spec-writer)

### 1. upgrade-to-2026-5-4: Upgrade openclaw from 2026.3.14 to 2026.5.4
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** We are currently exposed to three confirmed or likely vulnerabilities in our version range: CVE-2026-43534 (CVSS 9.1, Critical — untrusted hook metadata escalated to higher-trust agent context, fixed in 2026.4.10), CVE-2026-42435 (shell injection via argv-level env var injection, fixed in 2026.4.12), and CVE-2026-42436 (auth bypass in browser snapshot/screenshot/tab routes, fixed in 2026.4.14). Upgrading to the current stable v2026.5.4 closes all three CVEs in a single rollout and additionally delivers gateway startup and plugin-loading performance improvements. Rollout must follow the ADR-0001 order (labs -> admins -> ovk) with canary and restore-state probe active per ADR-0002; the labs tenant memory footprint should be validated before promotion given its proximity to the memory limit.
**Source:** https://github.com/openclaw/openclaw/releases ; https://www.thehackerwire.com/openclaw-input-validation-vulnerability-escalates-untrusted-input/ ; https://www.redpacketsecurity.com/cve-alert-cve-2026-42435-openclaw-openclaw/ ; https://www.redpacketsecurity.com/cve-alert-cve-2026-42436-openclaw-openclaw/

### 2. baileys-supply-chain-audit: Audit and harden Baileys dependency against supply-chain attack
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** A malicious npm package named "Lotusbail" (56,000+ downloads) impersonates the legitimate whiskeysockets/Baileys library and exfiltrates WhatsApp authentication tokens, session keys, and all messages — with persistence that survives package removal until linked devices are manually revoked. We use Baileys for the WhatsApp Web channel, and session tokens are stored in S3 per ADR-0002; a compromise would directly affect `ovk` production customers. The fix is to verify our lockfile pins the correct package name and integrity hash, add a CI check that rejects any resolution of Baileys-adjacent package names, and document a runbook for linked-device revocation in the event of a confirmed compromise.
**Source:** https://www.bleepingcomputer.com/news/security/malicious-npm-package-steals-whatsapp-accounts-and-messages/

### 3. discord-js-dm-fix: Upgrade discord.js to 14.26.4 to fix uncached DMChannel message loss
**Category:** performance
**Impact:** 3
**Effort:** 1
**Rationale:** discord.js 14.26.4 (released May 1, 2026) fixes a bug in MessageCreateAction where incoming DMs in uncached DMChannels are silently dropped, causing missed messages on the Discord channel. We use discord.js and operate the Discord channel across all three tenants; this is a straightforward dependency bump with no breaking changes noted and low rollout risk. The fix is a single patch-level upgrade.
**Source:** https://github.com/discordjs/discord.js/releases
