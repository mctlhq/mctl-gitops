## Top-3 (for spec-writer)

### 1. upgrade-to-2026-4-27: Upgrade openclaw from 2026.3.14 to 2026.4.27
**Category:** security
**Impact:** 5
**Effort:** 3
**Rationale:** The deployed version 2026.3.14 is exposed to five unpatched HIGH CVEs (CVE-2026-41371/41349/41342/41352/41359, fixed in 2026.3.22–2026.3.31), covering privilege escalation, agentic consent bypass, auth bypass, and node-pairing RCE. The target 2026.4.27 (released April 29) also includes the bearer-token echo fix from 2026.4.26, OpenTelemetry expansion, Telegram/Slack reliability improvements, and manifest-first plugin startup hardening — all of direct relevance to our three tenants. The upgrade path follows the mandatory labs→admins→ovk sequence per ADR-0001, with s3-sync canary stop/restart per ADR-0002.
**Source:** https://github.com/openclaw/openclaw/releases; https://cve.threatint.eu/CVE/CVE-2026-41371; https://ccb.belgium.be/advisories/warning-3-high-severity-vulnerabilities-openclaw-can-lead-rce-patch-immediately

### 2. axios-supply-chain-pinning: Audit and pin Axios transitive deps against WAVESHAPER.V2 backdoor
**Category:** security
**Impact:** 4
**Effort:** 2
**Rationale:** UNC1069 (North Korea-linked) backdoored Axios 1.14.1 and 0.30.4 with a RAT that exfiltrates browser credentials, npm tokens, and GitHub tokens — precisely the secrets our tenants hold (S3 credentials, channel OAuth tokens, GitHub deploy keys). While node-slack-sdk is already pinned to axios@1.15.0, the workspace-wide transitive dependency graph has not been audited. A full npm audit + lock-file pin + allowlist check prevents silent credential theft across all three tenants.
**Source:** https://thehackernews.com/2026/04/unc1069-social-engineering-of-axios.html

### 3. gateway-ui-heartbeat-canary-noise: Mitigate upstream #73836 heartbeat poll noise to prevent false s3-sync canary alerts
**Category:** performance
**Impact:** 3
**Effort:** 2
**Rationale:** Upstream bug #73836 introduces heartbeat poll noise on top of UI reconnect stalls and Telegram typing gaps. The heartbeat noise is a direct trigger for the false s3-sync canary alert footgun documented in ADR-0002 ("canary noise must be fixed, not silenced"). For ovk (high SLA) a false canary alert can mask a real S3-sync failure, leading to auth loss on the next pod restart. Cherry-picking the upstream fix (or applying a targeted workaround) and tightening the canary threshold can be done independently of the full version upgrade.
**Source:** https://github.com/openclaw/openclaw/issues (issue #73836); context/decisions/0002-s3-state-with-canary-and-probe.md
